# R2 第一条真实主路径

## 任务

把结衣的第一条真实主路径接通：成长领域 → 阶段目标 → 当前实践 → 完成/中断 → 回归，并保持旧行动兼容。

## 变更文件

- `backend/app/db/models.py`：新增成长领域、阶段目标、实践事件，并为旧行动增加可空阶段目标关联与实践状态。
- `backend/app/db/repositories/refactored.py`：新增关系和实践事件读写。
- `backend/app/services/jieyi/growth_path_service.py`：新增成长地图、创建和回归编排。
- `backend/app/api/jieyi/routes.py`、`backend/app/schemas/__init__.py`：新增产品 API 契约。
- `backend/migrate_growth_path.py`：幂等新增迁移；迁移前数据库已保留备份。
- `packages/jieyi-web/src/pages/Way.tsx`：道页创建成长领域和阶段目标、读回成长地图。
- `packages/jieyi-web/src/pages/Action.tsx`：行页选择阶段目标，支持当前实践完成/中断/回归。
- `packages/jieyi-web/src/action-focus.ts`：统一本地日历日期计算。
- `packages/jieyi-web/src/styles/global.css`、`shared/types/jieyi.ts`、`shared/api/services/jieyi.ts`：页面样式、类型和产品入口。

## 关键决策

- 只新增正式关系，不迁移或改写旧行动；旧 `schedules_new` 行动的 `stage_goal_id` 保持为空也能继续使用。
- 完成、中断、回归既更新当前状态，也写入 `practice_events`，因此中断不是清零，回归可以读回同一实践的轨迹。
- 凌晨日期使用本地日历，不再用 UTC 日期把今天的实践写到昨天。
- 初次验收不写个人内容；钧钧随后明确确认第一组建议后，由 Codex 通过正式产品入口写入领域、阶段目标和当前实践。
