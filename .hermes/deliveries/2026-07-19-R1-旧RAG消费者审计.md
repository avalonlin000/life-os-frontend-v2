# 2026-07-19 结衣 R1：旧 RAG 消费者审计

## 结论

旧 `knowledge-rag.service` 不能现在停用。结衣新产品主链已经没有直接运行时消费者，但 8768 实际是小雪知识检索的共享旧入口。

## 已核对的消费者

- 小雪 workbench 的 `GET /api/tk/search` → `tk_knowledge.service.search()` → `repository.search_rag()` → `POST http://localhost:8768/api/search`。
- 小雪 `lol_data/libs/embedding_engine.py` 的 `search_tk()` 被赛前/赛后分析路径调用，直接 POST 到 8768。
- 小雪知识写入/维护脚本调用 `POST /api/reindex`，维护报告调用 `GET /api/stats`。
- 小雪平台还保留指向 8768 的 TK 图谱、知识面板和分析页面入口。

## 运行证据

- `knowledge-rag.service` 为 active，主进程监听 `0.0.0.0:8768`。
- `GET http://127.0.0.1:8880/api/tk/search?q=BLG&limit=3` 返回 HTTP 200 和 TK 结果。
- 同一核对窗口的 RAG 日志出现本机 `POST /api/search 200`；`GET /api/stats` 返回 200。
- 结衣 `life-os-frontend-v2` 与 `workspace/hermes-refactor/backend` 生产代码未发现直接调用 8768。

## 本轮决策

- 不停止、不重启、不改 8768 配置，避免小雪 TK 搜索和分析静默降级。
- 将旧 RAG 从“结衣待停 shadow”改记为“小雪仍依赖的共享旧入口”。
- 后续退役前置条件：迁移小雪消费者到 MemPalace/新知识入口，完成搜索结果回读、重建链路和失败态验证，再停用旧服务。

## 风险备注

服务当前绑定 `0.0.0.0:8768`，日志中可见外部扫描请求；本轮仅完成消费者审计，未扩大范围处理公网暴露。
