# 结衣知行思产品层拆分：R1 第二切片交付记录

日期：2026-07-18

## 本次目标

把结衣的知、行、思主链从巨型 Agent 的直接数据库逻辑中拆出，保持现有页面和 API 入口不变。

## 已完成

- `KnowledgeService`：知识列表、导入、详情、分页。
- `ActionService`：今日行动读取、创建、完成/修改。
- `ReflectionService`：活动、心情、趋势、已生成每日复盘读取。
- `FeedbackLoopService`：思考卡、思考卡转行动、每日修炼、修炼勾选、今日复盘、明日调整写入。
- `ContextService`：每日计划、活动、心情、复盘、未完成行动、最近知识和目标的统一上下文。
- `PrinciplesService`：区分已验证 wisdom 与待验证方法论，保留来源和验证状态。
- `TodayService`：装配知、行、思、道首页 flow/aggregate，主入口不再直接调用 Agent 聚合。
- `DailyPlanService`：计划写入和行动投影继续作为唯一产品写入入口。
- 路由已切换到产品 Service；旧 Agent 方法保留为兼容入口，未改变外部 API 形状。
- 新增 Repository 查询边界，避免路由直接拼数据库查询。

- 没有执行真实计划刷新/写入，避免未经确认修改业务数据。
- 下一切片继续拆原则/今日聚合和 AI 能力适配，仍保持旧入口兼容。

## 产品影响

钧钧实际使用的知、行、思页面和入口不变；这次先把内部责任边界理顺，使知识、行动、复盘可以分别验证和继续替换，不会因为某一块 Agent 逻辑出问题而拖垮整个主链。
