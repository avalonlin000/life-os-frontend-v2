# 版本契合与纯阵容24场景

本文件只服务十英雄阵容推演。它允许使用比赛版本知识，但不允许引入队伍、选手、实时经济、赔率或赛果。

24场景覆盖强开、反开、Poke、滚雪球、后期大核、边线牵扯、资源团阵地等常见比赛形状。

## 版本来源

先运行：

```bash
python3 /home/ubuntu/.agents/skills/lol-lineup-analysis/scripts/read_version_context.py --json
```

来源顺序：

1. 用户明确给出的比赛版本。
2. 当前对话中已经确认的赛事版本。
3. 正式通用版本数据库与版本 Wiki 的一致结论。
4. 三者都不足时写 `版本 UNKNOWN`，只用稳定英雄机制推演。

脚本返回 `conflict` 时，必须展示冲突状态，只使用数据库和 Wiki 都支持的通用结论。队伍战术、教练、纪律性和单队适配内容全部忽略。

## 版本契合检查

逐方展示以下项目，不写空泛的“更适合版本”：

- 体系：强开 / Poke / 反开 / 边线 / 正面前后排 / 混合体系。
- 英雄环境：当前版本明确加强、削弱、优先级或陷阱；没有可靠证据就写 UNKNOWN。
- 线权与资源：哪几路可能先动，能否转先锋、小龙、野区或推塔。
- 装备与伤害：主要伤害类型、前排处理、第一件与两件套窗口只讲可靠趋势，不编具体数值。
- 资源节奏：阵容需要抢第一轮资源、连续提速，还是允许多轮防守与发育。
- 体系克制：强开、Poke、反开之间的关系是否在这十英雄中真实成立，不能只按标签自动判胜。

## 场景定位输入

先填七项阵容条件，再筛场景：

| 条件 | 可用值 |
|---|---|
| 线权 | 蓝多 / 红多 / 分散 / UNKNOWN |
| 野区联动 | 主动入侵 / 抓边提速 / 发育控图 / 反蹲保护 / UNKNOWN |
| 启动 | 多启动 / 单启动 / 反手为主 / 双方缺启动 |
| 前排 | 稳定 / 条件 / 缺失 |
| 输出 | 单核 / 双核 / Poke / 持续 / 爆发 / 输出断层 |
| 清线防守 | 强 / 中 / 弱 |
| 第二路线 | 有独立替代赢法 / 只有主链 |

编号沿用旧24场景，条件全部改为纯阵容结构。主场景只能选一个；无法唯一定位时列候选及缺失条件。

## 24场景矩阵

| # | 标签 | 纯阵容条件 | 典型走势 | 30分钟影响 |
|---:|---|---|---|---|
| 1 | 碾压局 | 一方多路线权、多启动、能连续推塔控资源；另一方清线与替代线都弱 | 前期滚雪球后单向推进 | 明显偏小 |
| 2 | 控图翻盘 | 先拿线权的一方缺可靠终结，另一方有稳定反手和资源团结构 | 前期被压，资源团反转 | 偏大或放弃 |
| 3 | 温水煮蛙 | 一方有线权、稳定控制和持续资源转换；另一方能守但难主动反击 | 连续失去视野、塔和资源 | 偏大，防守弱时偏小 |
| 4 | 强胜不碾压 | 主动方结构更全，但防守方有清线、反手或远程劝退 | 优势持续但推进缓慢 | 偏大 |
| 5 | 伪线优爆冷 | 线权方缺前排或稳定启动，另一方正面反打更完整 | 线优无法转团战，首个资源团反转 | 放弃 |
| 6 | 肌肉局 | 双方控制少、伤害高、保护弱 | 拼技能命中与集火，波动很大 | 放弃 |
| 7 | 无抓手局 | 双方启动、追击和留人都不足 | 清线、试探、难以连续击杀 | 偏大 |
| 8 | 野区结构差 | 一方打野需要主动入侵或抓人，但线上难先动；另一方能先支援 | 野区入口被压，第一轮资源失守 | 看清线；通常偏小或放弃 |
| 9 | 回合决胜 | 双方都有完整开团与反打，差别在赢团后能否立刻转资源 | 一波换一波，资源转换定走势 | 放弃 |
| 10 | 体系对抗 | 单挂下路与游走体系对上稳定正面五人组 | 一方扩张地图，一方逼正面 | 通常偏大 |
| 11 | 伪优翻盘 | 前期输出或线权领先，但缺控制、前排或第二轮；另一方曲线完整 | 前期看似舒服，中期开始失去接团权 | 偏大 |
| 12 | 无结构局 | 双方都缺稳定启动、前排与安全输出窗口 | 谁失误谁输，阵容难主动决定 | 放弃 |
| 13 | 预制链胜 | 一方启动、承伤、输出、追击四环天然连贯；另一方需要多次高难配合 | 完整链反复兑现 | 清线弱则偏小 |
| 14 | 控制反压 | 一方稳定控制与抓边工具更多，另一方只有伤害没有安全启动 | 线权未必领先，但接战权长期占优 | 偏小或放弃 |
| 15 | 镜像发育 | 双方都有前排、控制和后排核心，前期都不易突破 | 多轮资源团后决胜 | 偏大 |
| 16 | 有线杀不死 | 一方多路线权但缺留人、前排处理或推塔保护 | 线推过去却无法扩大优势 | 偏大 |
| 17 | 线权转后期 | 一方前期先动，另一方后期核心和防守更稳 | 前期资源与后期曲线赛跑 | 冲突时放弃 |
| 18 | 锚点断裂 | 一方唯一发动机被明确反制，且没有独立第二启动或第二输出 | 主体系无法开始 | 对手能转资源则偏小 |
| 19 | 三路博弈 | 双方优势分散在不同路线，支援与换边相互制约 | 没有单一主战场，频繁交换 | 偏大或放弃 |
| 20 | 回合效率决胜 | 双方结构均衡，谁赢团后更能推塔、拿大龙或连续追击成为核心差异 | 均势到一次高价值回合 | 放弃 |
| 21 | 单核断裂 | 一方只有一个安全输出点，且对方能稳定限制或逼退该核心 | 第一轮输出被断后无第二伤害 | 对手终结强则偏小 |
| 22 | 下路战略放弃 | 下路组合偏自保或单挂，辅助更适合游走，中野承担发动职责 | 下路让资源，中野扩大地图 | 通常偏大 |
| 23 | 线权养打野 | 多路线权能先支援打野入侵、控河道和抢第一轮资源 | 线上主动权反哺野区 | 转塔强则偏小 |
| 24 | 全图熄灯 | 打野需要主动做事但多路线权不足，阵容又缺清线与反手 | 野区、河道和防御塔连续失守 | 明显偏小 |

## 场景输出规则

完整展示：七项输入判断 → 主场景 → 候选场景 → 排除两个最像但不成立的场景。场景只描述比赛形状，不单独重复加权；最终强弱仍由版本契合、核心链、反制链、四阶段和终结能力共同决定。
