# Life OS 交付记录模板

> 每次小白完成开发/修复/部署/排查后，生成一份 delivery。小白保留全量；结衣和小雪只读各自可见摘要，不吃小白完整会话，也不展开对方项目细节。

## 任务

- 一句话说明这次做了什么。

## 变更文件

- `path/to/file`：改动说明
- 可由 `pnpm hermes:summary` 自动列出本次改动文件；人工只需补充关键文件的改动说明。

## 关键决策

- 为什么这么做。
- 有哪些边界没有做。
- 脚本会写入默认同步边界说明；人工补充本次任务的关键产品/工程决策。

## 验证结果

- 命令：`实际执行的命令`
- 结果：通过 / 失败 / 阻塞
- 证据：关键输出、URL、截图或日志路径
- `--verify` 示例：`命令：pnpm build:jieyi；结果：通过；证据：vite build completed in 1.23s`

## 风险与遗留

- 无 / 具体风险

## 可见范围

| 对象 | 可见内容 |
|------|----------|
| 小白 | 全量任务、变更文件、关键决策、验证、风险、后续动作 |
| 结衣 | 只看「给结衣」摘要和结衣知行合一相关影响；小雪电竞人生细节不展开 |
| 小雪 | 只看「给小雪」摘要和小雪电竞人生相关影响；结衣知行合一细节不展开 |

## 给小雪

- 她需要知道的数据、日报、查询点。
- 如果不需要同步，写“无”。
- 不写结衣知行合一的页面、接口、数据库、部署细节；最多一句“结衣侧有更新”。

## 给结衣

- 她需要知道的复盘、整理、陪伴、用户解释点。
- 如果不需要同步，写“无”。
- 不写小雪电竞人生的赛程、队伍、TK、交易记录、数据库、部署细节；最多一句“小雪侧有更新”。

## 后续动作

- 无 / 下一步明确动作
