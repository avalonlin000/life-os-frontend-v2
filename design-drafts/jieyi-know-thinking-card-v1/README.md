# 结衣知页 · 思考卡 v1

## 设计立场

知页不是知识列表，而是“结衣拿一条知识逼用户发生一次思考”。

## 核心 UX

1. 首屏只给一张主思考卡，避免刷列表。
2. 每张卡固定包含：内容、结衣问、结合你、建议、你的判断。
3. 新知和复习都进入同一种思考卡，只是来源和语气不同。
4. 导入降级为弱入口，主导入发生在和结衣的对话/后台。
5. 底部导航只保留单字：知 / 行 / 思 / 道。

## 视觉参考

- Claude / Anthropic：暖纸感、低压、有人味。
- Readwise Daily Review：每天处理少量知识。
- Anki：复习队列，但不做考试感。

## 取舍

强：
- 信息少但密度高。
- 每条知识都逼用户结合自己。
- 不像知识库/导入后台。

弱：
- 不适合一次浏览大量知识。
- 对后端生成质量要求高：必须能生成个人关联和问题。
