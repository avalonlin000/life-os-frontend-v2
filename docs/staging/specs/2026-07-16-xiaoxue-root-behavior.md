# 小雪根入口行为收口

## Goal

让小雪在真实日常会话中稳定听懂当前意图，只走一条正确路径；事实不确定时先核实；长流程没有真实完成证据时不宣布完成。

## Scope

- 统一小雪日常入口、纯阵容分析、钧钧交易系统、搜索、阶段知识整理和阶段交易复盘的优先级。
- 以当前纯十英雄阵容 Skill 为阵容分析唯一方法；交易系统单独消费阵容结果，不把队伍/盘口事实混入纯阵容判断。
- 增加最新意图防火墙、事实核验门和真实完成门禁。
- 清理或标注会继续被发现/执行的旧口径、影子入口和冲突测试。
- 增加真实用户原话驱动的路由/行为验收，覆盖首个动作、目标 Skill、事实边界和完成语义。

## Out of scope

- 不重写小雪人格，不增加业务能力，不更换模型。
- 不删除历史会话、历史交付或正式交易/知识数据。
- 不在业务会话中修改 Skill、SOUL、路由、能力地图或工作流状态。

## Contract

- 当前用户意图优先于历史任务、压缩摘要和默认流程；“先聊/不要动手/只问状态/把情况给我”禁止执行性扩展。
- 路由顺序固定为：阶段轨道 → 明确纯十英雄阵容 → 明确交易系统（非纯阵容）→ 搜索/数据 → 日常入口；混合请求按显式组合顺序串行，不允许两个能力竞争首步。
- 纯阵容分析只使用十英雄、等经济、正常发育、同等操作水平，输出倾向、阶段变化、比赛画像和观赛信号；不读取队伍、选手、TS、三维、盘口或赛果。
- 关键实体未核实或阵容不完整时，必须先核实或明确停下，禁止凭记忆补全。
- 长流程只有在真实正源写入、读回和必要镜像验证通过后才能进入完成态；校验失败只能停在恢复态，不能手改终态。
- “verified” 必须包含全新真实会话的路由和用户可见结果证据，静态文本检查不能单独放行。

## Failure behavior

- 规则或事实冲突：停在当前安全位置，说明冲突和缺失，不补故事、不假完成。
- 目标 Skill 不明确：只问一个改变结果的澄清问题，不先扫描、写入或派活。
- 旧会话/旧 Skill 描述过期：新会话刷新后再验收，旧上下文不作为新行为证据。

## Acceptance

- 真实新会话覆盖“先聊/只问状态/把情况给我/纯 BP/交易系统/未知英雄/阶段整理/失败长任务”八类原话。
- 每个场景记录首个工具动作、实际加载 Skill、用户可见首步、事实核验和完成/停下结果。
- 现有阵容、阶段工作流和相关产品文档测试全部通过；冲突旧测试必须被更新为当前契约而不是删除。
