# 小雪独立 TK 压缩与阶段整理解耦

## Decisions

goal: 钧钧可以随时单独让小雪压缩普通 TK；阶段知识整理复用同一压缩能力，不再复制压缩、归档或完成规则。

contract: 新增独立入口 `xiaoxue-tk-compression`。用户说“压缩 TK / TK 太多了 / 整理旧 TK / 清理 TK”时只进入该入口；用户说“阶段知识整理 / 更新版本理解 / 更新画像”时进入 `xiaoxue-knowledge-loop`；交易回溯仍只进入 `xiaoxue-stage-trading`。

contract: 独立压缩默认处理当前 TK 区内全部尚未归档、非 `compressed`、非 `trading_note` 且 `created` 合法的普通 TK。用户给出日期、队伍或阶段时按该范围收窄；不依赖“上次日期”，避免较晚导入但 `created` 较早的 TK 永久漏掉。

contract: 用户可见正式正文严格保持 2026-06-13 的三元组：`实体 | 对象 | 评价`。对象必须是可复用维度；评价必须是简短的强弱、稳定性、风格、变化或定位标签，不得写成比赛事件摘要。时期、逐条证据、覆盖账本和恢复信息只在后台审计包中保存，不污染正文。

contract: 每批只展示一次完整压缩预览。钧钧修改后重做并完整重显；只有原话“这批压缩通过”才允许正式写入。

invariant: 每份范围内的合格普通 TK 必须恰好进入“已压缩来源”或“明确暂缓”之一；禁止只抽取少数材料后把整批标为完成。排除项必须有机器可检查的原因。

invariant: 正式写入顺序固定为：生成压缩文件 → 读回正文 → 重建索引 → 按每个实体实际检索命中 → 原子归档被替代原文 → 再次确认当前 TK 区只剩压缩文件与暂缓项。任一步失败都不归档原文，也不进入完成态。

invariant: 原始 TK 永不删除。成功压缩的原文移入 `tk_archive`；错误压缩产物移入知识库之外的恢复备份，不允许继续污染当前或归档检索。

invariant: 日常与工作台的普通 TK 检索默认只搜索当前 TK 来源；归档 TK 只能显式选择后查询，避免已替代原文继续和压缩结论竞争。

invariant: 状态只能由确定性脚本推进并带完整性校验。业务会话不得手改状态、包、凭证或终态；完成凭证必须由真实文件读回、索引命中和归档对账共同生成。

contract: `xiaoxue-knowledge-loop` 保留版本、队伍、选手和画像差分判断，只在队伍整包确认后把获准的普通 TK 范围交给独立压缩执行器；它不再拥有正文渲染、覆盖统计、归档或完成实现。

failure: 资料不足、实体归属冲突或评价无法稳定归纳时列为暂缓并保留原文；不得硬编强弱标签。

failure: 旧状态无效或完整性损坏时创建受校验的恢复批次，保留现场并隔离错误产物；不得修补旧状态制造完成。

test: 路由测试证明“压缩 TK”只进入独立入口，“阶段知识整理”只进入阶段入口，两者不会竞争首步。

test: 契约测试证明事件汇总、长段评价、缺证据、部分覆盖、提前归档、假检索凭证、手改完成态和交易 TK 混入均被拒绝。

test: 临时知识目录真实演练完整走通预览、确认、写入、读回、索引命中、归档与失败回退；随后用全新小雪会话验证真实原话。

## Current recovery batch

scope: 重做 `2026-06-14~2026-07-14` 当前普通 TK；覆盖队伍和关键选手，不限于此前错误生成的五支队伍。

recovery: 隔离 2026-07-18 错误生成的五份压缩文件，保留恢复副本；无效的 `msi-2026-loop-v1.json` 保持原样作为事故证据，不再作为新批次状态。

gate: 新模块和路由通过验证后先生成完整预览；本次项目开工确认不替代内容确认，正式归档仍等待“这批压缩通过”。

## Deferred

deferred: 不恢复日报自动运行，不修改交易复盘，不扩展其他电竞项目，不重做知识导入与 RAG 引擎。
