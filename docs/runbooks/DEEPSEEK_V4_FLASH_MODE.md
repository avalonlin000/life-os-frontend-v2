# DeepSeek V4 Flash 专项交付模式

> 仅当钧钧明确要求“DeepSeek V4 Flash 托管/解析/云端极简代码”时适用。默认项目开发仍按当前仓库技术栈执行。

## 目标

- 代码面向 DeepSeek V4 Flash 解析和托管，优先清晰、扁平、可逐行理解。
- 交付覆盖必要的数据库结构、后端读写逻辑和前端展示逻辑。
- 完成后仍要走真实构建、接口和用户路径验证。

## 约束

- 零新增外部依赖：不新增 npm 包、第三方模块或 CDN。
- 前端优先 Vanilla JS + 原生 CSS；适合时使用单文件 `index.html` 或极简结构。
- 避免复杂 Class、高阶函数和深层异步嵌套；保持函数短、流程直、命名明确。
- 数据库优先 SQLite，提供清晰建表语句和最小 CRUD。
- 不因为本模式存在而绕开结衣 React/Vite 或小雪 FastAPI/Vite 的既有主项目结构，除非 Goal 明确要求新建独立极简应用。

## 安全边界

- 真实业务数据写入、数据库迁移、系统配置、网络、账号、密钥和生产 cron 仍需钧钧确认。
- 不把示例数据伪装成真实数据，不用静态页面假装后端已经实现。
