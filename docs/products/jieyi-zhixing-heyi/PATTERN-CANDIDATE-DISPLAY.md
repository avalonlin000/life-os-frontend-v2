# P3-4 反复模式候选前端展示

更新日期：2026-07-06

## 结论

JY-P3-4 已完成。`/reflect` 和 `/way` 均可展示 deterministic detector 输出的模式候选。

## 展示范围

- `/reflect`
  - 展示候选数量、状态、建议调整。
  - 展示证据日期、证据文本和写回目标。
  - 数据不足或无候选时展示明确空状态。
- `/way`
  - 新增“反复模式候选池”。
  - 候选标记为 `candidate`，不进入长期原则。
  - 展示模式类型、证据日期、关联行动、证据文本。

## 验收边界

- 不把候选伪装成已验证原则。
- 不在无真实证据时展示假候选。
- 后端 API 异常时显示不可用状态。
