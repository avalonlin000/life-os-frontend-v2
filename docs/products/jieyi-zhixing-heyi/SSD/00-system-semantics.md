## 0. V4 顶层语义

```text
人生方向筛选现实课题
  → 认识世界：事实 / 知识 / 理解 / 问题
  → 改造世界：方法 / 时间 / 实践
  → 反馈：结果 / 阻力 / 新问题
  → 世界观更新 / 方法论更新
  ↺
```

- 现实课题是知识、判断、方法、实践、反馈和更新的共同对象。
- 世界观回答“现实是什么、为什么会这样”；方法论回答“怎样更有效地改变现实”。
- 知行思道不是四个平行系统；它们只是支持上述循环的历史入口。
- 成长领域、阶段目标和当前实践不再是最高分类，只在用户需要时作为方向和实践背景。

## 0.1 三层运行架构

```text
后台认知层
  长期事实 / 行为 / 状态 / 语言 / 实践结果
  → 带证据且可推翻的工作假设
        ↓
结衣 Agent
  自然理解 / 主动提醒 / 解释规律 / 建议现实验证
        ↓
前端工作台
  阅读认知与知识 / 运行方法与实践 / 快速记录 / 结果验证 / 纠错确认
```

三层只是不同职责，不是不同事实源。所有正式内容仍归属现实课题、知识、方法、实践、反馈、世界观或方法论更新。

## 0.2 长期观察与推断契约

| 状态 | 定义 | 能否直接改变产品正式状态 |
|---|---|---|
| 观察信号 | 可核对的表达、行为、状态、时间、实践或结果 | 否；只作为原始证据 |
| 工作假设 | 结衣基于多条信号提出的可能解释，必须保留依据、反证和不确定性 | 否；不能冒充事实、正式认识或人格结论 |
| 主动提醒 | 当假设与当前现实课题相关且有足够证据时，在自然对话中指出矛盾或风险 | 否；提醒本身不是用户确认 |
| 现实验证 | 为区分不同解释而提出的一次可承担实践或观察 | 重要实践需用户自然确认后进入正式状态 |
| 已确认认识/方法 | 经用户确认，并继续接受现实结果检验的当前判断或做法 | 是；但仍可被后续实践修正 |

- “真正状态”不是系统可直接读取的字段；结衣只能通过长期证据提高判断质量。
- 荣格、阿德勒、斯多葛等可以作为生成假设的内部观察镜头，不得直接成为用户标签、诊断结论或页面模块。
- 单次晚睡、中断、情绪表达或任务未完成不得单独推出深层人格判断。
- 深层观察只有转化为更准确的认知、值得学习的内容、更有效的方法或现实验证时才算产品价值。
- 用户可以在对话或前端纠正观察与关联；修正后必须保留来源和更新历史，不静默改写过去。

## 1. 项目一句话定义

> 结衣知行合一是一套以长期成长为目的的人生反馈系统：帮助用户面对真实生活，在多个成长方向中选择当下可承担的实践，在中断后重新回归，并从行动反馈中逐渐形成自己的判断与原则。

## 1.1 北极星与系统立场

结衣帮助用户逐渐成为“有能力承担现实的人”：看得清事实，分得清边界，做得出选择，承担得起结果，也能持续修正自己。

系统立场：

- 情绪是事实信号，不是人格结论，也不是未经判断的最高指令。
- 结果不完全可控；用户的判断、选择和实践属于自己的责任范围。
- 坚持定义为回归能力；中断不自动清零，回归不等于补偿性追赶。
- 理念不作为页面内容或心理学诊断，而作为系统判断与语言边界。
- Agent 可以提出建议，但不能替用户改变方向、暂停阶段目标或提升正式原则。
- 前端不展示笼统的人格评价；后台工作假设只有在解释当前课题、知识选择、方法或实践时才按必要范围出现。

## 1.2 能力总图

```text
人生方向 → 现实输入 → 当前选择 → 实践与回归
                         ↓
                    反馈与整理
                         ↓
                    认知成长
                         ↓
                    节奏调整
```

知 / 行 / 思 / 道是用户入口层，不是互相孤立的功能模块：知承接输入，行承接实践，思承接反馈，道承接长期方向与原则。

交互层不是新的能力分类：结衣 Agent 负责模糊语境、推理和自然协商；前端负责持续展示、长内容、高密度操作、快速记录、搜索比较、纠错确认和连续历史。二者必须回到同一能力总图。

## 2. 核心语义

| 术语 | 定义 | 禁用替代 |
|------|------|----------|
| 认知资产 | 从行为、知识、反馈中沉淀出的可复用经验、判断、模式 | 内功、经验库 |
| 今日整理 | 对当天知识、行动、活动、备注的结构化总结，不是心理诊断 | 日报、复盘 |
| 节奏建议 | 基于反馈信号给出的次日计划密度、任务颗粒度、恢复/执行倾向建议 | 优化建议、效率分析 |
| 任务阻力 | 完成困难、反复重开、颗粒度过大、时机不合适等执行信号 | 拖延分析、懒散 |
| 执行队列 | 今日或选定时段要完成的任务列表 | 待办清单 |
| 活动记录 | 用户真实发生的活动和备注 | 心情日记 |
| 成长领域 | 用户长期愿意经营的方向 | 标签、人生 KPI |
| 阶段目标 | 某一阶段准备验证的具体变化 | 永久目标、硬性承诺 |
| 当前实践 | 为阶段目标服务、可由用户承担和验证的最小行动 | 普通待办 |
| 观察信号 | 关于现实或用户状态的可核对材料，只能作为后续认识的证据 | 人格特征、真实内心 |
| 工作假设 | 结衣用于解释信号的可修正候选，需保留依据和不确定性 | 诊断、结论、正式原则 |
| 回归 | 中断后基于当前现实重新进入方向和实践 | 补齐欠账、连续打卡 |

当前三层关系已由 `growth_domains → stage_goals → schedules_new.stage_goal_id` 承载；实践状态和完成/中断/回归轨迹分别由 `practice_status` 与 `practice_events` 保存。旧行动允许暂时没有 `stage_goal_id`，但不计入成长主路径。

## 3. 知 / 行 / 思 / 道 四层语义

### 知：认知输入层

```text
功能：外部知识导入、保存、标签、拆行动、深度学习。
用户动作：粘贴文本 / 输入主题 → 保存 / 学习 → 可选拆解为任务。
数据展位：知识列表、知识卡片、深度学习入口。
语义：外部世界 → 当前成长方向中的问题和待验证认识。不是收藏，是可进入实践的材料。
```

### 行：执行层

```text
功能：当前实践、完成/中断、降档、回归和手动补充动作。
用户动作：查看当前实践 → 用户确认 → 执行或记录中断 → 选择回归方式。
数据展位：实践卡片、可控范围、退阶动作、回归入口和反馈提示。
语义：阶段目标转成可承担实践 → 实践产生反馈信号。
```

### 思：整理反馈层

```text
功能：活动记录、统一复盘输入、今日整理、节奏建议。
用户动作：补记真实活动 → 写一段今日复盘 → 触发今日整理 → 查看节奏建议。
数据展位：活动列表、整理结果卡、建议卡、认知资产候选卡。
语义：事实与解释分开 → 识别阻力和有效策略 → 反向影响后续实践与节奏。
```

### 道：长期沉淀层

```text
功能：成长领域、阶段目标、认知候选、正式原则和长期模式。
用户动作：确认/调整方向 → 查看阶段进展 → 审核候选 → 对照原则与当前实践。
数据展位：成长地图、阶段目标、候选证据、正式原则和回归轨迹。
语义：短期实践持续服务长期方向；道是方向与原则的罗盘。
```

## 5. 每日整理输出结构

| 部分 | 内容 | 数据来源 |
|------|------|----------|
| 今日总结 | 今天做了什么、记录了什么、完成了什么、哪些没推进 | knowledge / actions / activities |
| 有效推进 | 哪些动作有用，哪些活动带来推进，哪些知识转成行动 | 完成的任务 / 拆解记录 |
| 节奏风险 | 是否过载、疲劳、输入多行动少、反复重开、计划密度太高 | 重开次数 / 导入量 / 活动量 / 统一复盘原文 |
| 后续计划建议 | 明天偏执行 / 偏整理 / 偏恢复 | 节奏风险评估结果 |
| 认知资产 | 可复用经验、反复主题、有价值判断、对后续有用的模式 | 整理过程中提取 |

## 6. 反馈调整规则

| 触发条件 | 动作 | 优先级 |
|----------|------|--------|
| 统一复盘原文出现明显疲惫 / 焦虑 / 拖延 | 次日偏恢复，只保留一个必须动作 | P0 |
| 反复重开任务（同一任务重开≥2次） | 建议拆小、换时间、降低密度 | P0 |
| 导入知识多但行动少 | 少导入，多转化 | P1 |
| 活动记录很多但总结少 | 次日偏整理，减少高压任务 | P1 |
| 完成顺畅 | 保留类似低阻力动作形式 | P2 |
| 多条件同时触发 | 高优先覆盖低优先 | — |

## 7. 文案禁用词 / 推荐词

### 禁用

- 内功库
- 荣格分析、阴影人格、人格类型
- 心理诊断、诊断、治疗
- 商业建议、投资建议
- 你失败了、你拖延了
- 积分数值、徽章、成就称号
- 复盘（用“今日整理”替代）
- 效率、生产力、自我优化

### 推荐

- 认知资产
- 今日整理
- 节奏建议
- 反馈信号
- 后续计划调整
- 任务阻力
- 降低密度
- 拆小动作
- 少导入，多转化
- 偏执行 / 偏整理 / 偏恢复
