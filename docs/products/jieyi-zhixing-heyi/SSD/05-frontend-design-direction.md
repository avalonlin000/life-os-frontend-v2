# 结衣知行合一 — 前端工作台设计方向 v2

> 版本：4.1 | 2026-07-22
> 状态：产品方向、功能去向和最终前端总图已批准；正式采用「安静文档」四页线框，现有五入口继续兼容，正式页面实现待开始。

---

## 核心定位

前端是结衣私人系统的可视化运行工作台。它围绕现实课题，把“认识世界 → 改造世界 → 反馈更新”长期展开并方便操作。

前端不是：

- 结衣 Agent 的表单版本。
- 展示“结衣如何评价钧钧”的人格分析页。
- 普通知识库、任务清单、情绪仪表盘或哲学内容站。
- 与聊天、后台并列的第二套人生系统。

## 与后台认知层、Agent 的关系

```text
后台认知层：观察长期信号，形成带证据、可修正的工作假设
        ↓
结衣 Agent：自然交流，主动指出矛盾，解释规律并提出建议
        ↓
前端工作台：阅读认知与知识，运行方法与实践，记录结果并纠正系统
```

- 后台可以使用心理学和哲学视角生成可能解释，但不输出人格评分或诊断。
- Agent 处理模糊表达、上下文理解、挑战和协商，不把对话做成逐字段填表。
- 前端处理长文字、高密度操作、快速或批量记录、搜索比较、精确编辑、结果验证和重要确认。
- 三层必须使用同一现实课题、知识、方法、实践和反馈正源。

## 前端必须服务的内容

### 现实课题

- 当前不满、希望改变、事实、主要矛盾和客观条件。
- 当前采用的认识、方法、实践和最近结果。
- 这一轮循环正在补哪一环，以及实践如何修正原认识与方法。

### 认识世界

- 当前课题值得理解的客观规律。
- 相关知识、长材料、学习顺序、来源和待验证问题。
- 已被实践支持、修正或推翻的认识。

### 改造世界

- 当前方法、具体步骤、时间安排和现实实践。
- 完成、中断、降档与重新回来。
- 每次实践需要观察的结果和下一轮调整。

### 反馈更新

- 真实结果及其与原方法、实践的关系。
- 世界观与方法论更新候选、确认状态和连续历史。
- 可搜索、可比较、可纠错的已验证知识和方法。

## 输入设计原则

- 前端必须保留高效率输入，不能因为 Agent 是日常协作者就降为只读档案。
- 长文粘贴、批量补记、结构化状态、时间安排和直接状态操作优先使用前端。
- 模糊经历、需要理解的问题、深层矛盾和方法协商优先使用 Agent。
- 快速记录先保存原始内容，再由系统建议关联；不得要求用户先选择事实、认识、方法、反馈或知/行/思/道分类。
- 正式认识、方向、重要实践和原则需要用户确认；确认可以来自自然对话或明确页面操作。

## 展示设计原则

- 长文字是主要内容形态之一，适合认知解释、学习材料、方法步骤和阶段整理；不强制拆成碎片卡片。
- 首屏优先展示与当前现实课题直接相关的认知、方法、实践和结果，不展示人格评价。
- “为什么推荐”只展示与当前决策必要的事实和依据，不倾倒后台推理草稿。
- 心理学、哲学和长期模式只影响系统判断，不直接成为页面分类或人物标签。
- 视觉继续采用暖夜纸感、移动端优先、少装饰、清楚来源和明确操作反馈。

## 现有入口审计保护原则（已执行）

当前 `/reality`、`/know`、`/act`、`/reflect`、`/way` 是已验证兼容入口；它们先接受下方功能审计，最终目标导航已经在本文后半部分另行批准。

功能审计逐项使用以下判断：

```text
保留在前端
合并到现实课题
转为 Agent 交互
转入后台自动处理
仅保留历史查看
删除重复入口
```

审计与设计期间遵守：

- 不删除现有能力和正式数据。
- 不新增第六个平行页面。
- 不根据现有 React 文件边界倒推产品结构。
- 不再把“知 / 行 / 思 / 道四页”视为最终前端结论。

## 2026-07-22 现有功能去向审计

判断标准：功能必须直接帮助看清现实、吸收与运用知识、形成方法、组织实践、读取结果或修正认识与方法。仅展示结衣如何评价用户、要求用户先完成系统分类、与现实课题无关的固定内容，不进入主工作台。

### 现实课题 `/reality`

| 现有功能 | 去向 | 结论 |
|---|---|---|
| 当前现实、希望改变、主要矛盾、客观条件 | 保留前端 | 现实课题是总纲规定的组织中心；适合持续查看和精确纠错 |
| 新建、编辑、切换、暂停、继续、完成课题 | 保留前端 | 重要状态需要可见、可控；Agent 也可协助讨论，但不能成为唯一入口 |
| 当前调用知识、综合判断、方法、实践、反馈和知识缺口 | 保留前端 | 必须展示私人知识怎样介入现实，不能只展示模型生成的结论 |
| 分别填写事实、理解、未知问题、方法的四组表单 | Agent 主导＋前端确认 | 事实可由快速记录进入；理解、问题和方法由结衣协助整理，前端只保留候选查看、纠错和确认 |
| 搜索并调用已有知识 | 现实页展示结果＋认识页深入比较 | 知识调用是现实课题形成方法的必经步骤，不再只是可选关联 |
| 建立实践、完成、中断、回归 | 合并到改造世界 | 状态按钮适合前端直接操作；实践内容必须说明服务哪个课题和方法 |
| 选择实践并记录真实反馈 | 合并到现实课题 | 保留快速反馈和明确关联；不要求用户填写世界观、方法论更新表 |
| 世界观/方法论更新候选确认 | 保留前端 | 属于用户主权和正式资产确认；Agent 可解释，前端负责集中核对 |
| 因果关系、事件轨迹、连续历史 | 保留前端 | 聊天无法持续展开全部关系，这是前端不可替代的价值 |
| 数量统计 | 仅留历史 | 只用于快速定位，不作为成长评分或主内容 |

### 知识 `/know`

| 现有功能 | 去向 | 结论 |
|---|---|---|
| 知识列表、来源、标签、搜索与长内容阅读 | 保留前端 | 适合持续展示、搜索和阅读；必须能说明与现实课题或长期认识的关系 |
| 长文、文章、聊天和想法导入 | 保留前端 | 长文本粘贴比聊天高效；先保存原文，关联与分类由系统协助 |
| 深度学习材料、三问、学习包和五卡 | 保留前端 | 长时间阅读与结构化学习适合页面；从固定 P0 主流程降为按需工具 |
| 今日一问 | 合并到认识世界 | 只有与当前现实课题或正在学习的问题相关时展示；通用问题不占主入口 |
| 把今日一问转成行动 | Agent 主导＋前端确认 | 先形成方法或现实验证，再进入当前课题实践；不直接制造普通任务 |
| 知识“拆解 → 行” | 合并到现实课题 | 改为提出与当前课题有关的方法/实践候选，不再把知识机械拆成通用行动 |
| 三组静态内容样例 | 删除重复 | 已完成引导使命，与私人真实内容无关，继续展示会制造噪声 |
| 刚保存材料单独提示卡 | 保留前端 | 只作为短暂保存回执，不升级为独立模块 |

### 行动 `/act`

| 现有功能 | 去向 | 结论 |
|---|---|---|
| 当前主行动/当前实践 | 保留前端 | 是改造世界的直接操作面，必须持续可见 |
| 完成、中断、重新回来 | 保留前端 | 高频状态操作比聊天更快，且符合“坚持是回归能力” |
| 手动补充或替换今天的一件事 | 保留前端 | 页面输入高效；正式进入主循环时必须关联现实课题或方法 |
| 请求行动建议 | Agent 主导＋前端确认 | 建议需要结合长期上下文和当前矛盾，前端展示建议与确认，不承担推理 |
| 行动阻力信号独立板块 | 转后台 | 结衣在后台判断；前端只在它改变当前方法或实践时展示必要依据，不展示对用户的评价 |
| 今日学、今日复习 | 合并到认识世界 | 属于知识线，不应夹在行动页中打断实践 |
| 固定“今日修炼” | 仅留历史 | 已经进入当前现实课题的方法保留；与课题无关的固定打卡不进入主工作台 |
| 知识拆解行动、今日做、其他行动三组列表 | 合并到改造世界 | 按当前课题、当前实践和其他未关联记录重新组织，不再按旧来源分成三套任务 |

### 反馈整理 `/reflect`

| 现有功能 | 去向 | 结论 |
|---|---|---|
| 活动开始、补记、结束 | 保留前端 | 计时和精确补记适合直接操作；活动是现实事实，不是人格评价 |
| 一段式今日复盘输入 | 保留前端 | 长文字和批量补记适合页面，同时允许自然聊天输入同一正源 |
| 今日整理长文 | 保留前端 | 是事实转成下一轮认知和方法的重要结果，适合长期阅读 |
| 手动“生成今日整理”主按钮 | 转后台 | 正常情况由固定整理流程完成；前端只保留失败后的明确重试，不让用户维护系统 |
| 今日对账、完成数量等概览 | 仅留历史 | 只作为事实摘要，不作为自律评分或首页核心 |
| 反复模式、10 天趋势、节奏风险独立板块 | 转后台 | 结衣负责综合判断；只有结论会改变学习、方法或实践时才展示必要证据 |
| 后续调整 | 合并到改造世界 | 直接进入当前方法或下一项实践，不单独形成建议墙 |
| 认知资产候选 | 保留前端 | 进入统一确认区，保留来源、证据和确认状态 |
| 静态复盘样例 | 删除重复 | 与个人真实记录无关，且会干扰自然输入 |

### 长期方向与原则 `/way`

| 现有功能 | 去向 | 结论 |
|---|---|---|
| 成长领域、阶段目标和方向概览 | 保留前端 | 属于低频长期资产和现实课题筛选背景，不占日常首屏 |
| 新增领域、阶段目标表单 | Agent 主导＋前端确认 | 方向需要讨论和解释，不适合用两个输入框草率建立；前端保留纠错和正式确认 |
| 正式原则列表与来源 | 保留前端 | 是经实践验证的长期认知资产，适合阅读、搜索和对照 |
| 认知资产候选池 | 保留前端 | 合并到统一确认区，不再作为独立页面主线 |

| 原则来源说明 | 合并到长期资产 | 每条原则就地展示来源和验证，不单独占用说明板块 |
| 静态原则候选样例 | 删除重复 | 已完成演示作用，不能与真实候选并列 |

## 审计后的目标功能面

功能审计先收出四个功能面和一个跨页面入口，随后已在下方最终总图中确定用户侧名称和顺序：

1. **当前现实课题**：现实问题、调用的私人知识、综合判断、方法候选、准备验证的实践和知识缺口。
2. **认识世界**：真实知识来源、不同知识的组合与冲突、适用边界、方法候选、新知识转化和深度学习。
3. **改造世界**：经确认方法的来源、触发条件、步骤、观察指标、反例、完成/中断/回归和真实结果。
4. **长期资产**：知识使用记录、等待检验项、认识/方法更新候选、已验证个人方法、知识缺口和连续历史。
5. **全局快速记录**：在任何位置写入现实事实、新知识或实践结果；先保存原文，再提取可确认的关联和方法结构。

反馈不是新的第五套系统：即时结果回到当前现实课题，行动调整进入改造世界，经过验证的认识与方法进入长期资产。

## 最终前端总图（已批准目标）

用户侧导航统一为：

```text
现实 / 认识 / 实践 / 积累
              ＋
        全局「记一笔」
```

- **现实**是默认首页和现实课题中心。
- **认识**回答“现在应该理解什么、学习什么”。
- **实践**回答“现在采用什么方法、具体怎么做”。
- **积累**保存经过反馈检验的长期成果、确认项和历史。
- **记一笔**不是第五个内容系统，而是贯穿全部页面的快速输入入口。

```text
┌──────────────────────────── 结衣前端 ────────────────────────────┐
│ 当前现实课题：正在认识和改变什么                     [记一笔] │
├──────────────┬──────────────┬──────────────┬────────────────────┤
│    现实      │    认识      │    实践      │       积累         │
│ 默认首页     │ 认知与学习   │ 方法与行动   │ 验证成果与历史     │
├──────────────┴──────────────┴──────────────┴────────────────────┤
│ 原始记录 / Agent 对话 / 页面操作                                │
│          ↓                                                     │
│ 当前课题 → 认识与知识 → 方法与实践 → 真实结果 → 认识/方法更新 │
└────────────────────────────────────────────────────────────────┘
```

## 页面一：现实

### 核心问题

> 面对当前现实问题，私人知识库提供了什么，以及为什么形成现在这个方法？

### 页面内容顺序

1. **现实问题**：当前发生什么、希望改变什么、客观条件和需要区分的问题。
2. **本次调用的私人知识**：真实知识标题、来源、为何与当前问题相关；测试材料和无关内容不得展示。
3. **综合判断**：不同知识怎样共同解释现实，明确区分知识原意、结衣推断和仍未确认的部分。
4. **适用边界**：现有知识能解释什么、不能解释什么，防止用心理或哲学解释覆盖客观问题。
5. **知识形成的方法候选**：显示知识到方法的转化过程，用户确认或修改后才能进入实践。
6. **准备怎样验证**：本轮实践、观察对象和结果标准；不把一次盈亏或单纯完成视为有效。
7. **知识缺口**：私人知识不足时明确显示需要补什么，不调用模型常识伪装已有知识。
8. **完整转化过程**：现实 → 知识 → 判断 → 方法 → 实践 → 反馈 → 更新，课题管理和时间线默认收起。

### 不再出现

- 没有私人知识来源的通用建议。
- 只显示一段“当前认知”却不说明知识怎样参与。
- 知识库完整列表、全部活动、模式大盘和原则池。
- 对用户状态或人格的长期评价。

## 页面二：认识

### 核心问题

> 私人知识库中的不同知识怎样共同分析当前现实，并形成可检验的方法？

### 页面内容顺序

1. **当前调用的知识**：逐条显示真实标题、来源、核心方法和在当前课题中的作用。
2. **知识组合**：说明各条知识分别解释哪一层问题、共同点在哪里、怎样互补或冲突。
3. **适用边界与反例**：明确哪些现实不能由当前知识解释，哪些判断仍需事实或专业知识。
4. **方法候选**：从知识组合中形成针对当前现实的方法，保留来源链并等待用户确认。
5. **知识缺口**：显示当前还缺哪类规律或方法，参与后续学习和导入排序。
6. **全部知识库**：搜索真实方法、适用问题、来源和边界；不与当前调用结果混成普通列表。
7. **新知识转化**：保存原文后提取问题、方法、适用条件、边界和可验证动作，用户确认后参与以后检索。
8. **深度学习与认识历史**：按需查看一条知识怎样被理解、用于现实、被实践支持、修正或推翻。

### 展示形态

- 以文章、来源、比较、边界和转化关系为主，不强制拆成小卡片。
- 明确标识私人知识、外部补充知识和结衣推断，三者不能混写成一个答案。
- Agent 的心理学或哲学推理只有在私人知识或明确外部来源支持、且与现实问题相关时才展示。

## 页面三：实践

### 核心问题

> 经确认的知识方法怎样进入真实条件，并产生能够反过来检验知识的结果？

### 页面内容顺序

1. **本轮采用的方法**：说明它来自哪些私人知识、服务哪个现实问题、适用边界是什么。
2. **触发条件**：只有现实出现什么信号时才使用，避免把方法做成脱离场景的固定打卡。
3. **具体步骤**：把方法变成一次可执行的现实检验，保留完成、中断、回归和调整。
4. **观察指标**：说明真正要观察的现实变化，不用任务完成、分数或单次结果冒充方法有效。
5. **反例与失效条件**：同时记录方法没有解决什么，是否出现需要其他知识解释的结果。
6. **真实结果**：记录触发、执行、变化和与原方法预期的差异，自动回到同一现实课题。
7. **其他方法**：同一时期默认只突出少量方法，避免多项实践互相干扰；其他内容按需展开。

### 不再出现

- 今日学、今日复习等认知内容。
- 独立行动阻力分析板块。
- 知识拆解行动、今日做、其他行动三套来源列表。
- 与现实课题无关、没有适用条件的固定修炼打卡主线。
- 只问“完成了吗”却不记录方法是否改变现实。

## 页面四：积累

### 核心问题

> 现实结果怎样反过来支持、修正或淘汰知识，并形成更适合钧钧的个人方法？

### 页面内容顺序

1. **等待现实结果**：显示已经形成方法、但尚未完成真实检验的知识转化项。
2. **认识更新候选**：原知识对现实的解释是否得到支持，证据、反证和不确定性是什么。
3. **方法更新候选**：方法是否改变现实、适用条件和失效条件是什么；用户确认后才提升。
4. **已经进入个人系统的认识与方法**：保留来源知识、现实课题、实践证据、版本和边界。
5. **知识使用记录**：每条知识被哪些现实问题调用、形成过什么方法、结果如何。
6. **知识缺口**：现实暴露但当前知识库无法处理的问题，参与后续学习和知识导入排序。
7. **完整飞轮与历史**：记录 → 调用知识 → 形成方法 → 实践 → 反馈 → 更新的连续历史，以及长期方向和课题历史。

模式、趋势和节奏分析仍在后台运行；只有它们能连接到真实知识、现实证据和可验证方法时才进入候选。任何知识不能因为“听起来正确”直接提升为个人原则。

## 全局入口：记一笔

### 目标

让前端承担聊天不擅长的长文本、批量、精确和快速操作，同时不要求用户先理解系统分类。

### 默认交互

```text
点击「记一笔」
  → 一个大输入区，直接写或粘贴
  → 可选快捷动作：现实记录 / 新知识 / 实践结果 / 普通记录
  → 保存原始内容
  → 现实记录建议关联当前课题；实践结果建议关联原方法
  → 新知识提取问题、方法、适用条件、边界和可验证动作
  → 经用户确认后，知识才参与以后现实分析
```

- 默认不强迫选择分类；快捷动作只是提高已知场景的速度。
- 保存后先确认原文已经保留，不用弹出连续追问。
- 错误关联可以在当前内容旁直接纠正。
- 后续可以扩展附件或其他输入方式，但不在 M1 真实使用前扩张。

## 页面之间如何闭环

| 从哪里 | 用户动作 | 去哪里 |
|---|---|---|
| 现实 | 查看被调用知识怎样形成判断 | 认识 |

| 实践 | 记录触发、执行、结果和反例 | 现实课题历史＋积累候选 |

| 任意页面 | 记一笔 | 原文先保存，再按现实记录、新知识或实践结果进入同一转化链 |

正式实现统一采用「安静文档」：

## 正式视觉落地（2026-07-23）

低保真的“安静文档”信息结构继续有效；正式视觉在此基础上采用 Apple HIG 风格的产品叙事层，最终选择以下折中：

- 首屏与关键阶段可以使用接近一屏的沉浸式留白，帮助用户一次只聚焦一个问题。
- 复杂知识、表单、反馈证据、候选和历史继续保持单列连续阅读，不做强制横向轮播或营销式分页。

- 暖米色仍是结衣的基础人格，紫色—暖橙渐变只承担关键词、品牌和当前状态强调，避免变成冷白科技模板。
