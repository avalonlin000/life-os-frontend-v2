# 结衣知行合一 — 界面规格 (UI Spec)

> 基于 `/home/ubuntu/workspace/knowledge/wiki/结衣LifeOS/规格重补/ui-spec.md` 拆分整理
> 版本：4.1（前端工作台目标契约） | 2026-07-22
> 配色方案：暖夜 · 深棕 + 奶油

## V4 当前已实现课题入口（兼容基线）

- `/` 与 `/jieyi/` 打开当前焦点现实课题；规范路径为 `/reality`。
- 首屏顺序：当前现实与矛盾 → 当前认识 → 当前方法与实践 → 最近反馈 → 下一缺环。
- 无课题时用两个自然语言输入分别保存“现在怎样”和“希望怎样”，不把两者混成一段当前现实。
- 不做无关卡片大屏；首屏只突出当前方向和唯一下一步，认识 / 实践 / 反馈一次只展开一个工作段。
- 反馈必须明确选择对应实践；页面直接展示方法 → 实践 → 反馈 → 认识更新 / 方法更新的因果关系。
- 关联知识支持搜索完整已读列表；读取失败与真正空库必须使用不同状态。
- 连续历史展示真实事实、候选、确认、拒绝、实践事件和反馈，不再只显示数量统计。
- 课题设置可以编辑名称、当前现实、希望改变、主要矛盾和客观条件；暂停/完成必须二次确认。
- 课题库按进行中、已暂停、已完成分组；无当前焦点时先让用户选择继续旧课题或另开课题。
- 实践卡可展开开始、中断、回归和完成轨迹；知/行/思/道页保留返回当前现实课题的轻入口，并记住离开前所在阶段。
- 知识、行动、复盘和原则入口继续可达，但都必须能回到当前课题。

上方内容记录 2026-07-21 已验证实现，不自动代表最终信息架构。2026-07-22 产品审计已经确认：五个平级入口仍是过渡状态；在下一轮前端重整前继续兼容，不据此扩张更多平行页面。

## V4.1 前端工作台目标契约（Planned）

前端的唯一产品职责是把现实课题的“认识世界 → 改造世界 → 反馈更新”持续展开并方便操作。它不是结衣 Agent 的表单版，也不是展示结衣如何评价用户的分析面板。

### 必须承载

| 用户需要 | 前端结果 |
|---|---|
| 长时间阅读 | 与当前课题相关的认知、知识、长材料、学习顺序和来源 |
| 知道怎么做 | 当前方法、步骤、时间、实践、降档与回归方式 |
| 高效率输入 | 长文粘贴、快速或批量补记、结构化状态、直接完成/中断/回归 |
| 看实践是否有效 | 现实结果、方法→实践→反馈关系、认识与方法更新 |
| 查找和比较 | 搜索、时间线、历史课题、已验证知识与方法 |
| 保持用户主权 | 纠正错误关联，查看候选依据，确认方向、正式认识、重要实践和原则 |

### 禁止承载

- 人格评分、心理标签、“结衣眼中的你”或情绪状态大屏。
- 将荣格、阿德勒、斯多葛等理论直接做成用户画像模块。
- 把后台工作假设和推理草稿不加筛选地倾倒给用户。
- 要求用户为了记录先判断内容属于事实、认识、方法、反馈或知/行/思/道。
- 与现实课题无关的无限知识信息流、普通任务清单或连续打卡体系。

### 输入原则

- 前端保留输入能力；“Agent 是日常协作者”不等于“所有输入只能聊天”。
- 日常主路径只要求三类主动输入：保留原文、填写当前实践、记录真实结果。事实纠错、手动知识关联、主题替换、深度验收和更换行动都保留，但默认收起，不能与主输入并列索取。
- 同一事实只允许一个正式编辑入口：实践结果在现实页写，实践页只负责执行与回到现实页；行动列表只保留一个完成控件。
- 适合模糊表达、需要理解、挑战和协商的内容优先由 Agent 处理。
- 适合长文本、批量、精确编辑、选择、搜索和直接状态操作的内容优先由前端处理。
- 快速记录可以先保存原始内容，再由系统建议关联；不得强迫用户先完成系统分类。
- 正式认识、方法、方向、重要实践和原则继续保留用户确认，确认可以来自自然对话或明确页面操作。

### 信息架构约束

- 现实课题仍是组织中心。
- 认识世界、改造世界和反馈更新是内容结构，不等于必须各自新增一个平行页面。
- 现有 `/reality`、`/know`、`/act`、`/reflect`、`/way` 在专项信息架构审计前保持兼容；旧入口是否合并、降级或重命名必须根据真实使用路径决定。
- 下一轮设计先完成“保留 / 合并 / 转入 Agent / 转入后台 / 删除”的功能审计，再画页面；禁止先按现有文件结构继续堆 UI。

## V4.2 最终目标导航与页面职责（Planned）

```text
现实 / 认识 / 实践 / 积累
         ＋ 全局「记一笔」
```

| 导航 | 首要问题 | 主要内容 |
|---|---|---|
| 现实 | 现在正在认识和改变什么？ | 当前课题、关键认知、当前方法/实践、最近反馈、下一缺环、候选与轨迹 |
| 认识 | 现在应该理解和学习什么？ | 认知长文、推荐材料、知识搜索、待验证问题、深度学习、认识历史 |
| 实践 | 基于当前认识，具体怎么做？ | 方法步骤、时间、当前实践、活动、完成/中断/回归、快速反馈 |
| 积累 | 哪些认识和方法值得长期保留？ | 待确认、已验证认识/方法、每日与阶段整理、课题历史、方向和连续历史 |

- 「记一笔」贯穿全部页面，默认先保存原文，再由系统建议关联；用户不必预先分类。
- `/reflect` 与 `/way` 从目标一级导航退出，但旧链接继续兼容；正式数据分别重组到现实、实践和积累。
- 手机端四项底部导航；桌面端同一顺序，不增加另一套大屏信息架构。
- 完整页面顺序、跨页关系和旧入口迁移见 `SSD/05-frontend-design-direction.md`。

## 0. 能力总图对 UI 的约束

四个入口共同承载一条主循环：

```text
方向 → 当前实践 → 真实反馈 → 回归/调整 → 认知成长
```

页面不展示斯多葛或阿德勒术语，也不把哲学做成独立卡片；这些理念只体现在信息顺序、确认边界和失败/回归状态中。

## 3. 知页 `/know`

### 区块顺序

1. 深度学习
2. 外部知识
3. 个人记忆 / 整理洞察
4. 每日小学习
5. 导入入口

### 深度学习面板

| 区域 | 规格 |
|------|------|
| 输入区 | 主题输入框 + 范围下拉 + “生成学习包”按钮 |
| 资料结果区 | 显示真实 Wiki 材料；fallback 时只能显示“搜索接口未就绪” |
| 深挖问题区 | 3 个问题卡片，点击后高亮 |
| 学习包区 | 核心材料 / 相关旧知识 / 边界候选 |
| 五卡引导 | problem / structure / relation / boundary / action 五个 textarea |
| 验收回写 | quality level + destination + “回写验收”按钮 |

状态规则：

- fallback：黄色徽章，“接口未就绪 · 显示占位提示”；不能算真实学习完成。
- live：绿色徽章，“API 已连接”；材料必须来自真实 Wiki 检索。

## 4. 行页 `/act`：当前实践与回归

### 区块顺序

1. 当前最值得承担：首屏只突出一个当前实践；用户手动添加优先于知识拆解、AI 建议和只读计划。
2. 换成自己的一件事：用户可以直接写一句当前实践；写入成功后成为新的主实践。
3. 关联阶段目标：已有阶段目标时，新增实践先选择服务于哪个阶段目标；没有阶段目标时仍可添加兼容行动，但页面明确提示尚未进入成长主路径。
4. 回归状态：挂接阶段目标的实践支持“完成今天这个”“今天先中断”“重新回来”；中断不清零，回归和完成都展示真实状态。
5. 其他内容：今日学、复习、修炼、阻力信号、退阶动作和其余实践默认收起，需要时再展开。

### 日常降噪

- 行页不得展示 `xiaobai-smoke`、接口验证、单元测试、原始密钥报错等工程记录。
- 阻力信号如果来自上述非个人行动记录，前端直接过滤，不把系统维护噪声解释成用户的人生阻力。
- 完成主行动后，重新读取真实数据并选择下一条；页面不批量伪造“今日必须完成”。
- 实践中断后显示回归入口；不自动生成补偿性任务，不把中断解释为方向清零。

### 今日学

- 展示 `dailyPlan.learn`：pillar、title、content、questions、source。
- 展示 `dailyPlan.review`：fromDate、title、snippet、question。
- 无数据时显示“今日学习内容尚未生成（凌晨4:05自动生成）”。

### 今日做

- AI 建议任务显示在其他行动；知页拆解仍单独归类。
- 用户添加任务显示在“今日做”。
- 完成后：opacity 降低、line-through、按钮变为已完成。
- 再点已完成任务可重开。

## 5. 思页 `/reflect`

### 区块顺序

1. 活动记录
2. 今日复盘统一输入
3. 今日整理
4. 节奏建议
5. 历史整理 / 认知资产候选

### 今日复盘统一输入

- 前台只保留一个 textarea，不再拆“今日备注 / 明日调整 / 心情 / 思想”。
- 用户只打一段自然语言，保存到 `mood.note`。
- 保存复盘不创建 schedule，不影响 /act 今日行动队列。
- 长期模式分析先依赖约 10 天原文沉淀，不在前台增加分类选项。

### 今日整理

- 已有 `daily_review` 时展示 summary、highlights、concerns、suggestion。
- 无 review 时显示“还没有今日整理”与生成按钮。
- 生成中按钮 disabled，文案“生成中...”。
- 生成失败不能填充 mock 数据。

## 6. 道页 `/way`

`/way` 是当前「道」页主入口；`/dao` 仅保留为兼容 alias 或历史称呼，进入后跳转到 `/way`。底部导航和侧边导航都使用 `/way`，不要改成 `/dao`。

### P1 页面结构（成长地图）

1. 成长领域与阶段目标
2. 当前实践关联
3. 认知资产候选与来源
4. 反复模式与正式原则

道页不是 P0 闭环的阻塞项，但导航和空状态应该保留，避免用户误以为系统断层。

## 8. 视觉系统：产品叙事式安静文档（2026-07-23）

本轮采用“关键阶段沉浸、复杂内容连续阅读”的折中方案：不把结衣改成营销落地页，也不退回平均用力的卡片堆。

### 全局规范

- 页面继续使用单列主路径；桌面正文宽度上限为 `980px`，顶部导航宽度上限为 `1040px`。
- 桌面端使用吸顶毛玻璃主导航；手机端使用轻量毛玻璃品牌栏和底部四项主导航。
- 现实、认识、实践、积累保持同一顺序；旧 `/reflect`、`/way`、`/dao` 统一显示为“积累”归属。
- 关键圆角使用 16 / 22 / 28 / 36px 四级连续半径，禁止同一页面混用大量无规律圆角。
- 表面使用分层柔和阴影，不使用粗黑描边和高对比硬阴影；毛玻璃必须同时保留可读背景色和细边界。
- 标题使用紧字距和自适应大字号；动态长任务标题自动降档，不能因追求视觉冲击损害完整可读性。
- 紫色到暖橙色渐变只用于品牌标记、状态强调和关键标题，不用于大面积正文。

### 页面节奏

- 现实首屏、实践主行动、旧复盘首屏和旧方向首屏使用约 620–720px 的沉浸式关键舞台。
- 认识页以当前课题私人知识为第一屏；“今日一问”作为下一段沉浸式认知入口。
- 积累页先用大留白说明长期价值，后续每个主要阶段形成独立阅读区。
- 手机端复杂表单、知识正文、历史和候选不强制撑满一屏，继续自然连续阅读。

### 可访问性与响应式

- 交互控件保持至少 44px 触控高度；键盘焦点使用清晰的紫色外环。
- 支持 `prefers-reduced-motion`，关闭页面入场和非必要过渡。
- 390px 视口不得产生横向溢出；桌面端不显示重复底部导航。
