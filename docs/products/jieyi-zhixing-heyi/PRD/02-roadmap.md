# 结衣知行合一 — Roadmap

> 版本：4.1（三层职责与前端重整路线） | 2026-07-22

> 当前路线以私人系统投资总纲为最高判断标准；下方 R2/P0-P3 内容保留为历史实现基线，不再自动代表当前投资优先级。

## 当前投资路线

### M0：总纲和组合确认（已完成）

- 确认结衣长期只服务钧钧，覆盖整个人生，以“认识世界＋改造世界”为最高主轴。
- 确认人生方向负责筛选、现实课题负责组织、认识与改造负责运行、实践反馈负责更新，结衣 Agent 是统一日常协作者。
- 已按新总图完成现有结衣项目与结衣 Agent 的宏观审计；旧 P0-P5 验收只作为历史实现证据。
- 确认外部项目组合：Rill + SwarmVault + Memex；复用 MemPalace；ActivityWatch 延后评估。
- 确认不以通用化、多人化或商业化为目标。
- 确认三层职责：后台认知层负责可修正观察，结衣 Agent 负责自然协作与主动提醒，前端工作台负责长内容、实践操作、快速记录和结果验证；前端不展示人格评价。

### M1：证明“认识世界 → 改造世界 → 再认识”的现实循环（当前最高优先级）

目标：围绕一个真实生活问题，先认识事实、矛盾和客观条件，再形成方法进入实践；用现实结果检验并修正原有认识和方法，完成一次完整循环。现状审计确认这是当前最大的产品缺口。

投资判断：如果学习和行动仍然彼此分离，后续知识图谱、主动发现、复杂自动化和页面扩张都不值得继续投入。

M1 交互验收必须同时证明：自然对话不退化为表单；长期观察只形成可验证假设；前端能稳定展示当前认知、学习材料、方法、实践和结果；聊天与前端写回同一条现实课题历史。

### M2：用知识线和岁月时间管理线承接两大系统

目标：知识线重点服务“认识世界”和世界观更新；岁月时间管理线重点服务“改造世界”和方法论落地；两条线通过实践反馈形成循环。

投资判断：系统开始从“偶尔有效”变成“持续可用”。

### M3：让反馈改造系统

目标：现实结果持续修订世界观、方法论和每条系统线，并调整结衣后续的认识重点、实践方式和线间协调。

投资判断：结衣从知识执行器升级为能够自我校准的私人系统。

### M4：形成全人生认识与改造复利

目标：不同人生领域的系统线能够共同帮助认识和改造现实；系统从长期实践中不断更新对世界和自身的认识，并形成越来越有效的方法论。

投资判断：结衣开始具备长期私人基础设施的复利价值。

## 投资优先级

1. 看清现实中的事实、矛盾和客观条件。
2. 形成能够改变现实的方法并进入实践。
3. 用实践结果检验和修正认识与方法。
4. 让知识线、时间管理线等系统线按现实需要协同发展。
5. 高级自动化、复杂分析和视觉包装。

## 当前暂不扩张

- 通用化、多用户、商业化和社区能力。
- 第二套知识库、第二套 Agent 入口或平行产品主线。
- 复杂任务管理、连续打卡、积分和大屏仪表盘。
- 在 M1 真实价值被证明前新增高级模块。
- 人格评分、心理标签、自我评价大屏，以及没有转化为认知或方法的“结衣眼中的你”。

## 当前路线：先统一能力，再改实现

### R2-1：统一产品总图

- 以“人生方向 → 现实输入 → 当前选择 → 实践与回归 → 反馈与整理 → 认知成长 → 节奏调整”为唯一产品能力骨架。
- 将陪伴、轻量整理、深度学习、每日计划、模式识别等归回对应能力，不再作为平行产品主线。
- 现有知 / 行 / 思 / 道与现实课题入口先作为兼容实现保留；最终导航必须重新按现实课题主循环审计，不能因为旧页面存在就默认继续五入口并列。

### R2-2：补齐方向到实践的关系

- 领域、阶段目标、当前实践三层语义必须能互相追溯。
- 从已有记录发现的方向只能进入候选，不能静默成为正式方向。
- 当前实践必须能说明正在服务哪个阶段目标，以及成功/中断后产生什么反馈。

### R2-3：建立可回归的日常主路径

- 每天只突出当前最值得承担的一项或少量实践。
- 状态差时降低实践强度，不默认取消长期方向。
- 中断后提供回归入口，不以连续天数或补偿性任务惩罚用户。

### R2-4：收口 Agent → 产品交接

- 后台认知层从长期事实与结果形成带证据、可修正的暂时假设，不声称掌握用户的“真正状态”。
- Agent 负责自然理解上下文、主动指出有证据的矛盾、提出建议、解释依据和在重要变化前取得自然确认。
- 产品负责保存方向、实践、事实、反馈、候选和正式原则。
- Agent 不直接改变长期方向，不静默暂停，不静默提升原则。

### R2-4A：重整前端工作台

- 前端围绕现实课题承载认识世界、改造世界和反馈更新，不展示人格评价或后台心理推理过程。
- 保留适合页面的长文字阅读、知识学习、时间实践、快速/批量记录、搜索比较、纠错确认和历史回看。
- 移除要求用户先完成系统分类的交互；事实、认识、方法、反馈的关联由系统和结衣协助完成。
- 在完成信息架构审计前不直接删除旧页面，不新增第六个平行入口。

### R2-5：真实使用后再调整

- 先走通“确认方向 → 选择实践 → 执行/中断 → 回归 → 反馈”的真实路径。
- 在真实使用形成足够证据后，再决定是否增加更多自动分析或高级调度。

## 历史实现基线（保留验收证据）

### P0：知行思闭环

1. /know 知识输入与列表稳定。
2. /know 深度学习入口可用：主题 → 搜索 → 三问 → 学习包 → 五卡回写。
3. /act 今日执行队列稳定：完成、重开、手动补充。
4. /reflect 活动记录与统一复盘输入稳定。
5. /reflect 今日整理和节奏建议显式成功/失败。

---

### P1：道层增强

1. /act 展示 daily_plan.review 复习卡片。
2. /way 展示长期目标；/dao 仅保留为兼容 alias/历史称呼。
3. /way 展示智慧卡片；底部导航和当前主入口仍使用 /way。
4. 反复模式识别后台方案：明确真实数据源、写入路径和约 10 天原文约束。
5. Hermes cron 接管每日整理生成。

---

### P2：认知资产沉淀

1. 定义认知资产候选结构：标题、内容、来源日期、原始复盘、关联行动、关联知识、状态。
2. 从 daily-review / reflection 中生成认知资产候选，不直接伪造长期原则。
3. 写入 Wiki 或原则候选池前保留来源记录和确认边界。
4. /way 展示来源可追溯的原则/认知资产候选。

---

### P3：反复模式识别产品化

1. cron 或后台任务读取最近 10-14 天 mood / activities / schedules / daily-review。
2. 运行 deterministic pattern detector，产出 `repeated_patterns`、`rhythm_risks`、`rhythm_suggestion`。
3. 写回 daily-review JSON 扩展字段或 `PatternCandidateModel` / Wiki 模式候选文件。
4. /reflect 或 /way 展示模式候选、证据日期、建议调整和候选状态。

---

## 开发节奏

- 先保闭环，再做美化。
- 先保真实数据，再做 AI 生成。
- 先移动端可用，再补桌面端展示。
- API 不可用时宁可显式失败，不做假成功。
