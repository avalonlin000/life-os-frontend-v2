# 结衣知行合一 — 产品需求文档 (PRD) / Overview

> 基于 `结衣 Life OS — 系统语义设计文档 (SSD)` 与 `prd.md` 拆分整理
> 版本：1.0 | 2026-06-28
> 优先级：P0（知行思闭环） / P1（道层增强）

---

## 1. 项目一句话定义

> 结衣知行合一是个人反馈调整系统——帮用户低负担记录真实行为，把知识转成行动，把行动反馈整理为认知资产，再反向调整后续计划。

---

## 2. 系统目标

| 序号 | 目标 | 说明 |
|------|------|------|
| T1 | 知识转行动 | 外部知识不是收藏，而是可拆解为具体行动的材料。 |
| T2 | 行动产反馈 | 执行、完成、重开、阻力都成为后续调整信号。 |
| T3 | 今日整理 | 多源记录生成结构化今日整理，不做心理诊断。 |
| T4 | 认知资产 | 有价值判断、经验、模式写入 Wiki 成为长期资产。 |
| T5 | 节奏校准 | 根据疲惫、过载、输入过多、重开等信号调整次日节奏。 |
| T6 | 移动端优先 | 手机端短反馈、快操作、强确认；桌面端只做补充。 |

---

## 3. 用户场景

### 核心场景：日常短反馈

用户打开结衣 → 查看今日执行队列 → 完成/重开任务 → 记一笔活动或备注 → 查看今日整理 → 离开。

### 补充场景：知识输入

用户看到有价值内容 → 粘贴到知页 → 保存为知识 → 可选拆成行动 → 进入行页执行队列。

### 补充场景：深度学习

用户输入粗主题 → 系统从本地 Wiki 搜索材料 → 生成 3 个深挖问题和学习包 → 用户完成五卡 → 回写验收。

### 辅助场景：长期回顾

用户查看道页 → 对照长期目标、智慧卡片、反复模式 → 校准近期行动。

## 4. 产品边界

| # | 不做什么 | 理由 |
|---|----------|------|
| B1 | 不做心理诊断 | 越权、技术不可靠、容易伤害用户。 |
| B2 | 不做重问卷 | 核心是轻记录，不是评分体系。 |
| B3 | 不做普通 todo list | 任务必须有反馈信号和节奏调整，不只是打勾。 |
| B4 | 不做普通知识库 | 知识必须能转行动或沉淀认知资产。 |
| B5 | 不做桌面大屏优先 | 结衣主要在移动端使用。 |
| B6 | 不和小雪混成一个电竞/生活大杂烩 | 结衣只负责知行思道闭环。 |
