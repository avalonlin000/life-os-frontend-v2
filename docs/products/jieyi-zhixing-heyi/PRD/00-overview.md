## 决策账本（2026-07-22 重新开工）

本节只负责区分“钧钧已确认的产品决定”“运行审计可核实的客观事实”“仍待论证的设计建议”和“已经否定的方向”。后续文档、线框和实现不得把后三类内容冒充为钧钧已经确认的需求。

### A. 已锁定的产品决定

1. 结衣长期只做钧钧的私人系统，覆盖整个人生，不面向多人或商业化。
2. 项目的最高收口是“认识世界＋改造世界”：因为对现实不满，所以持续认识客观规律、寻找方法、进入实践，并用结果修正认识和方法。
3. 记录、知识库、复盘、正反馈和飞轮不是彼此独立的模块；它们共同服务知识进入现实并形成持续进化。
4. 结衣的核心不是针对某个问题临时生成答案，而是形成一套固定的知识与现实交互方法；固定的是转化过程，不是固定答案。
5. 私人知识库里的有用知识必须能够在相关现实问题中重新被调用，参与分析并形成针对性方法；不能只是保存、阅读或等待钧钧自己想起来。
6. 实践既用于改造现实，也用于检验知识；“干中学、学中干”必须成为产品真实能力，而不是抽象口号。
7. 知识线与岁月时间管理线是两条不同系统线；先验证知识线，再接入时间管理线。多条人生方向允许长期并行，不预设数量上限。
8. 结衣可以动态判断当前重点、提出方法和实践建议，但只能建议；正式采用、改变方向和提升原则必须由钧钧确认。
9. “坚持”定义为中断后仍能回到长期方向的回归能力，不等于永不中断。
10. 结衣聊天应保持自然和潜移默化，不退化为表单；前端负责聊天不擅长的持续展示、长文字阅读、高效率输入、纠错、确认和历史回看。
11. 前端不展示人格评价、心理标签或“结衣眼中的我”；心理学和长期观察只能提高后台假设质量，最终仍由现实证据和实践检验。
12. 前端一级方向锁定为「现实 / 认识 / 实践 / 积累」＋全局「记一笔」，采用安静文档式单列阅读；聚焦工作台式双栏方案不采用。

### B. 当前可核实的客观事实

1. 结衣正式网页、产品 API 和旧五入口当前可访问；正式导航仍是现实 / 知 / 行 / 思 / 道，新四页尚未进入正式页面。
2. 现实课题已经支持建立、编辑、切换、暂停、完成和恢复；方法确认后可以建立实践，并记录中断、回归、完成和反馈。
3. 当前知识只能由用户手动搜索并关联到现实课题；系统尚不能根据现实问题主动调用知识、比较适用性、边界、冲突和知识缺口。
4. 反馈可以生成认识更新和方法更新候选，但确认后仍停留在当前课题，尚未真正更新长期私人知识或方法系统。
5. 正式产品库当前没有现实课题；第一件真实个人课题尚未开始持续使用。
6. 产品知识表当前有 12 条记录，其中 8 条是工程验收材料；排除验收材料后有 4 条真实内容。测试内容尚未从产品查询层统一隔离。
7. 结衣 Agent 已有受限产品交接设计和适配器，但尚未挂载到真实 profile；聊天入口目前不能完成整条产品写入和读回。
8. 安静文档四页目前只是低保真线框，不是正式产品实现。

### C. 尚未锁定、必须经过审计和产品论证的内容

1. 每条知识最终采用哪些固定字段和抽取规则。
2. 知识不足时是否、何时以及怎样引入外部新知识。
3. 多条知识怎样排序、组合、处理冲突并形成一个或多个方法候选。
4. 实践结果达到什么条件后，才能更新个人方法、世界观或正式原则。
5. 第一阶段选哪一个真实问题作为价值验证，以及需要持续多久。
6. 新四页的最终文案、具体区块、交互细节和桌面宽度。
7. 岁月时间管理线在知识线验证完成后怎样接入。

### D. 已否定、不再作为默认方向

- 只做记录、收藏、复盘或普通知识库。
- 用户说出问题后由模型绕过私人知识直接生成泛泛建议。
- 把知识机械拆成任务，或者用完成打卡代替现实检验。
- 用人格评价、情绪分数、心理标签或状态画像作为前端主要成果。
- 让钧钧自己跨多个页面拼装知识、方法、实践和反馈。
- 在产品主循环尚未走通前扩张高级分析、第二套入口或更多独立模块。
- 以聚焦工作台、卡片大屏或管理后台作为结衣前端的主要形态。

### E. 重新开工纪律

先完成真实能力审计和项目诊断，再定义第一阶段结果；第一阶段确认后才进入正式实现。低保真线框、页面改名、构建通过或局部接口存在都不能单独代表项目已经完成。

## 0. 私人系统投资总纲

### 0.1 投资命题

项目的根本出发点是钧钧对当前生活不满，并希望通过认识和实践改变现实。

结衣是一套长期只服务钧钧、覆盖整个人生的私人系统。它以“认识世界”和“改造世界”为最高结构：持续认识现实及其客观规律，持续寻找和实践改变现实的方法，并在实践中修正世界观与方法论。

### 0.2 产品承诺

> 帮助钧钧更准确地认识世界，更有效地改造现实，并在实践、认识、再实践、再认识的循环中持续增强能力。

### 0.3 核心飞轮

```text
现实状况与矛盾
        ↓
认识世界：观察事实、吸收知识、理解客观规律、更新世界观
        ↓
改造世界：形成方法、组织时间和行动、进入现实实践
        ↓
实践结果：变化、问题、阻力和新矛盾
        ↓
再认识：检验和修正原有认识与方法
        ↓
再实践：用新的世界观和方法论继续改造现实
        ↺
```

### 0.4 组织中心与顶层关系

结衣以“现实课题”为主要组织中心。现实课题指：当前生活中什么不符合长期方向、真实情况和主要矛盾是什么、准备怎样认识和改变它。

- 人生方向决定哪些现实课题值得长期投入。
- 现实课题承接当前真正需要认识和改变的事情。
- 人生领域用于查看全局，不拆成互不相干的独立系统。
- 目标和任务是改造现实过程中使用的工具，不成为产品中心。

```text
钧钧：人生方向、价值判断、最终确认
                  ↓
现实课题：当前不满、事实、矛盾、客观条件
                  ↓
认识世界：现实记录＋人类知识 → 当前世界观与判断
                  ↓
改造世界：方法论 → 时间、行动与现实实践
                  ↓
现实结果：变化、阻力、新事实和新问题
                  ↓
反馈更新：修正世界观与方法论，继续下一轮实践
                  ↺
```

知识线重点支持认识世界，岁月时间管理线重点支持改造世界，复盘和反馈连接两者。结衣 Agent 是整套系统的日常协作者与统一入口，不是第三套人生系统。

#### 0.4.1 固定的知识—现实转化方法

结衣的核心产品不是针对某一个问题生成一次建议，而是让任何现实问题都经过同一套可追溯、可复用、可修正的方法：

```text
大量记录现实
  → 识别当前问题、事实和客观条件
  → 从私人知识库调用相关规律与方法论
  → 比较适用条件、冲突、边界和知识缺口
  → 结合当前处境形成方法候选
  → 由钧钧确认后进入现实实践
  → 记录真实结果并复盘
  → 验证、修正或淘汰原有认识与方法
  → 回写个人系统，参与下一轮现实问题
```

- 固定的是转化过程，不是固定答案；同一问题可以因现实条件、知识积累和历史反馈不同而得到不同方法。
- 结衣必须说明调用了哪些私人知识、每条知识为何相关、共同形成什么判断、哪些地方仍然不知道。
- 私人知识库没有足够依据时，必须显式形成知识缺口，不得用模型常识冒充私人知识结论。
- 新知识保存后需要提取“解决什么问题、方法、适用条件、边界和可验证动作”，经用户确认后才获得参与现实分析的资格。
- 方法有效性不以“任务完成”或一次结果好坏判断，而以它是否在适用条件下改变现实、以及反例和失效条件是否被记录判断。

大量记录提供现实材料，知识库提供人类已有认识和方法，实践检验二者能否结合，复盘把结果重新写回个人系统。这条转化方法是记录、知识、正反馈和进化飞轮的共同收口。

#### 0.4.2 三层产品职责

结衣不是“一个聊天机器人加一组页面”，而是由三层共同运行的同一套私人系统：

```text
后台认知层：长期观察事实、行为、状态与结果，形成可修正的工作假设
        ↓
结衣 Agent：自然交流、理解上下文、指出矛盾、解释规律、提出建议
        ↓
前端工作台：持续阅读认知与知识，运行方法与实践，记录结果并修正系统
```

- 后台认知层服务于更准确的建议，不向用户输出人格评分或心理诊断。
- 结衣 Agent 是日常协作者，负责处理模糊、需要理解和需要讨论的内容；自然聊天不应退化成字段问答。
- 前端工作台是现实课题主循环的可视化运行界面，负责长内容阅读、高密度操作、快速记录、搜索比较、纠错确认和连续历史。
- 三层读取和写入同一套现实课题、知识、方法、实践与反馈，不建立聊天版、前端版或分析版三套事实源。

### 0.5 核心资产

- 对现实状况、矛盾和客观规律不断更新的认识。
- 吸收他人知识并经实践检验后形成的个人世界观。
- 在现实改造中验证、修订和积累的方法论。
- 实践过程、结果、问题和认识变化的连续历史。
- 结衣对钧钧长期经历、边界和有效方法的连续理解。
- 由长期信号、暂时假设、验证结果和修正记录组成的可修正个人模型；它是后台工作资产，不是前端人格评价。

### 0.6 产品形态

- **私人性**：长期只服务钧钧，不以通用化、多人化或商业化为目标。
- **全人生**：学习、工作、关系、状态、时间、选择和成长都可以进入，但统一归到认识世界或改造世界。
- **世界观持续更新**：知识库、记录、外部学习和复盘都服务于更准确地认识现实，不把任何一次认识固化为终极答案。
- **方法论持续更新**：岁月时间管理法及其他体系都是改造世界的方法线，必须在现实中运行、检验和修订。
- **实践第一**：不把认识和改造拆成两个封闭阶段；实践既是改造世界的方式，也是认识世界的来源和检验。
- **多条系统线**：知识线、岁月时间管理线和其他体系可以并行存在，由结衣在更高层协调。
- **用户主权**：结衣可以整理、建议和发现候选，但不能替钧钧确认真理、改变方向或宣布有效。
- **自然协作**：日常交流优先保持真人式上下文，不要求钧钧先把经历人工分类为事实、认识、方法或反馈。
- **前端工作台**：前端既不是只读档案，也不是 Agent 的表单替身；它重点承载与现实课题有关的认知、学习材料、方法步骤、时间实践、结果验证和高效率输入。
- **观察可修正**：结衣可以主动指出矛盾，但必须把深层判断当作待验证假设；心理学、哲学和长期模式只用于提高观察质量，不能包装成已经知道“真正状态”。

### 0.7 成功标准

项目是否值得继续投入，只看以下结果是否持续出现：

- 钧钧能够更清楚地看见当前生活的事实、矛盾和可改变部分。
- 外部知识能够帮助理解现实，而不是只增加收藏数量。
- 新认识能够形成方法并真正进入现实实践。
- 实践结果能够检验和修正世界观与方法论。
- 现实生活出现可感知的改善，钧钧认识和改造现实的能力持续增强。
- 多条系统线能够共同服务现实改造，而不会互相冲突或制造额外负担。
- 结衣的长期观察能够转化为更有用的认知、学习建议、方法或实践调整，而不是堆积对钧钧的评价。
- 聊天与前端各自发挥优势，但任何入口都不会要求钧钧重新拼装同一条认识与改造循环。

### 0.8 不投资的方向

- 不把结衣做成普通知识库、任务清单、连续打卡、情绪仪表盘或抽象哲学展示。
- 不为了功能丰富扩张平行页面、目标层级、工作区或第二套知识正源。
- 不把外部内容推荐做成无限信息流。
- 不把哲学、心理学或任何理念包装成必须学习的课程体系。
- 不为了未来可能的其他用户牺牲钧钧个人系统的深度和有效性。
- 不做人格评分、心理标签、“结衣眼中的你”或以情绪曲线为中心的自我评价大屏。
- 不把后台推理过程原样倾倒到前端；只有能改善认识、学习、方法、实践或验证的结果才进入用户工作台。
- 不让钧钧为了使用前端先学习系统分类；快速记录可以保持自然，正式认识、方法和方向仍保留确认边界。

### 0.9 外部项目吸收原则

Rill、SwarmVault、Memex、MemPalace 和未来可能接入的其他项目都只是材料来源。是否吸收某项能力，只问：

> 它能否帮助钧钧更准确地认识现实，或更有效地改造现实，并让实践结果继续更新认识和方法？

不能直接服务这条投资命题的能力，不进入结衣主系统。

### 0.10 阶段判断

```text
阶段一：证明一个真实生活问题能够走通“认识 → 方法 → 实践 → 再认识”
阶段二：证明知识线与岁月时间管理线能够分别服务认识和改造，并形成循环
阶段三：证明多条系统线能够长期协同，持续改善现实和更新世界观、方法论
```

在阶段一的真实价值被证明前，不扩张高级分析、复杂自动化或额外入口。

## 1. 项目一句话定义

> 结衣知行合一是一套长期只服务钧钧、覆盖整个人生，帮助钧钧持续认识世界、改造世界，并在实践中更新世界观和方法论的私人系统。

## 1.1 产品北极星

结衣最终帮助钧钧成为“有能力承担现实的人”：看得清事实，分得清边界，做得出选择，承担得起结果，也能把真正相信的东西逐渐做到。

坚持在本产品中定义为“回归能力”，不是连续打卡或永不中断。情绪可以提供信号、改变行动强度和时机，但不能未经判断独占人生方向。

## 1.2 已批准的完整产品顶层设计

```text
人生方向
    ↓ 选择值得改变的事情
现实课题
    ↓
认识世界 ──→ 当前世界观与判断
    ↓                 ↓
现实记录与人类知识   方法论
                      ↓
                  改造世界
                      ↓
                时间、行动与实践
                      ↓
                  现实结果
                      ↓
       反馈并更新世界观与方法论
                      ↺
```

钧钧是认识和改造世界的主体，保留人生方向、正式认识和重要行动的确认权。结衣项目保存并运行长期人生系统；结衣 Agent 负责理解、调用、建议、回看和协调，是日常交互入口。陪伴、轻量整理和复盘都是交互方式，不是平行产品主线。

三层职责不改变上述主循环：后台认知层持续提供可修正的观察，Agent 在对话中把观察转成问题、解释和建议，前端把已经需要长期阅读或操作的认知、知识、方法、实践和结果稳定展开。前端不得展示对钧钧的笼统人格评价，也不得只剩课程推荐和任务列表。

## 1.3 历史用户主循环（保留作实现基线，非当前产品总纲）

```text
确认成长领域
  → 选择阶段目标
  → 提出当前实践
  → 用户确认并执行
  → 记录真实反馈或中断
  → 生成回归/调整建议
  → 用户确认后继续实践或修正方向
```

## 2. 系统目标（按能力总图）

| 序号 | 目标 | 说明 |
|------|------|------|
| T1 | 方向承载 | 支持多个成长领域、阶段目标和当前实践长期并行存在。 |
| T2 | 现实输入 | 记录知识、行动、阻力、状态和生活事实，不以计划代替事实。 |
| T3 | 当前选择 | 把长期方向转成当下可承担、可验证、属于用户选择的最小实践。 |
| T4 | 回归能力 | 支持降档、中断和重新开始，不把一次中断解释为方向清零。 |
| T5 | 反馈成长 | 从实践结果中提炼认知候选，经用户确认后沉淀为正式原则。 |
| T6 | 动态节奏 | 根据真实反馈建议焦点、密度、方法和回归路径；重要改变由用户确认。 |
| T7 | 移动端优先 | 手机端短反馈、快操作、强确认；桌面端只做补充。 |

## 3. 用户场景

### 核心场景：从方向到实践

用户打开结衣 → 看当前成长焦点 → 确认一个当前实践 → 执行或记录中断 → 提交真实反馈 → 获得回归/调整建议。

### 补充场景：知识进入实践

用户看到有价值内容 → 关联一个成长领域或阶段目标 → 提炼待验证的问题 → 可选转成当前实践；不以收藏数量作为成果。

### 补充场景：深度学习

用户输入粗主题 → 系统从本地 Wiki 搜索材料 → 生成 3 个深挖问题和学习包 → 用户完成五卡 → 回写验收。

### 辅助场景：长期回顾与回归

用户查看成长地图、阶段目标、认知候选和正式原则 → 看到中断与回归轨迹 → 确认下一阶段的调整。

### 日常场景：自然交流与后台观察

用户自然讲述经历、状态或困惑 → 结衣结合长期记录识别事实、矛盾和可能解释 → 在必要时主动指出问题并提出现实验证 → 暂时判断继续等待证据，正式认识、方向和重要实践仍由用户确认。

### 前端场景：长内容与高效率操作

用户需要集中阅读认知与学习材料、粘贴长内容、快速补记多条事实、安排时间实践、搜索比较历史或纠正结衣理解时使用前端；所有操作继续围绕现实课题，不另建普通知识库或任务系统。

## 4. 产品边界

| # | 不做什么 | 理由 |
|---|----------|------|
| B1 | 不做心理诊断 | 越权、技术不可靠、容易伤害用户。 |
| B2 | 不做重问卷 | 核心是轻记录，不是评分体系。 |
| B3 | 不做普通 todo list | 任务必须有反馈信号和节奏调整，不只是打勾。 |
| B4 | 不做普通知识库 | 知识必须能转行动或沉淀认知资产。 |
| B5 | 不做桌面大屏优先 | 结衣主要在移动端使用。 |
| B6 | 不和小雪混成一个电竞/生活大杂烩 | 结衣只负责知行思道闭环。 |
| B7 | 不把哲学或心理学做成内容模块 | 相关理念只作为系统判断原则，不要求用户学习术语。 |
| B8 | 不替用户决定方向 | Agent 可以分析和建议；改变领域、阶段目标、暂停状态或正式原则必须由用户确认。 |
