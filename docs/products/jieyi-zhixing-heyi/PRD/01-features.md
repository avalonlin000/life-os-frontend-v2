> 基于原 `prd.md` 拆分

## V4 整体重做主路径

```text
现实记录与问题
  → 调用私人知识：规律 / 方法 / 条件 / 边界
  → 比较适配：共同点 / 冲突 / 缺口 / 当前判断
  → 形成并确认方法：针对当前现实，不是通用建议
  → 现实实践：时间 / 行动 / 中断 / 回归
  → 实践反馈：结果 / 反例 / 失效条件 / 新问题
  → 更新个人认识与方法论
  ↺
```

这是所有现实课题共享的固定工作法。产品不允许跳过私人知识调用，直接让模型生成泛泛建议；也不允许知识不足时伪装成已有结论。

| 能力 | 用户看到的结果 | 旧能力处理 |
|---|---|---|
| 现实课题 | 进入结衣先看到正在认识和改变的一件事 | 新的产品中心 |
| 认识世界 | 事实、知识、理解和问题有来源、有状态、可修正 | 复用知识与复盘能力 |
| 改造世界 | 方法明确服务哪个课题，实践支持完成、中断和回归 | 复用行动、活动和日程 |
| 实践反馈 | 结果能追溯到原认识与方法 | 复用思页、每日整理和趋势 |
| 世界观/方法论更新 | 两类更新分开成为候选，用户确认后沉淀 | 复用原则候选和确认边界 |
| 结衣 Agent | 读取、建议、确认、写入、回读同一课题 | 旧陪伴与整理保留为交互方式 |

完成标准：用户不再跨四页自行拼装；一件现实课题拥有连续历史；现实出现可感知变化；至少一项世界观或方法论被确认、修正或推翻。

## V4.1 三层交互职责（已批准产品定义）

| 层级 | 负责什么 | 不负责什么 |
|---|---|---|
| 后台认知层 | 汇总长期事实、行为、状态、语言与实践结果；形成带证据、可被推翻的工作假设；根据验证结果修正个人模型 | 不输出人格评分，不把心理学解释当诊断，不直接改变方向或正式原则 |
| 结衣 Agent | 保持自然对话；理解模糊输入；主动指出有证据的矛盾；解释值得学习的认知；提出方法、实践和验证建议 | 不把对话做成逐字段填表，不用一次迹象断言“真正状态”，不绕过用户确认 |
| 前端工作台 | 长文字认知与学习材料、方法步骤、时间和实践操作、快速/批量记录、搜索比较、结果验证、纠错和重要确认 | 不展示“结衣如何评价我”，不要求用户手工完成事实/认识/方法分类，不建立第二套主循环 |

### 长期观察如何产生产品价值

```text
可核对信号
  → 暂时假设（含依据、反证和不确定性）
  → 对话中的解释或主动提醒
  → 一次可承担的现实验证
  → 结果更新认识、方法和个人模型
```

- 两天晚睡、一次中断或一句情绪表达只能形成信号，不能单独成为人格或心理结论。
- 荣格、阿德勒、斯多葛等只作为后台提出可能解释的观察镜头；最终判断服从事实、现实条件和实践结果。
- 观察只有转化为当前认知、学习内容、方法选择或实践调整时，才成为用户可见成果。

### 前端必须产出的四类结果

1. **现实课题**：当前不满、事实、主要矛盾、客观条件与这一轮循环所处位置。
2. **认识世界**：当前值得理解的规律、相关知识与长材料、待验证问题、已被实践支持或推翻的判断。
3. **改造世界**：当前方法、具体步骤、时间与实践、中断后的回归方式、需要观察的结果。
4. **反馈更新**：实际发生的变化、原认识和方法是否成立、世界观/方法论更新与连续历史。

前端同时提供全局快速记录能力，适合长文粘贴、批量补记、结构化状态和直接操作；系统负责后续关联，用户不必先选择知/行/思/道分类。现有 `/reality`、`/know`、`/act`、`/reflect`、`/way` 继续作为兼容入口；它们已经完成去向审计，目标导航见下方 V4.3。

## V4.2 前端功能去向审计结论

2026-07-22 已完成现有五入口功能审计，目标不是删除前端，而是把功能从旧页面分类收回同一现实课题循环：

- **必须保留在前端**：现实课题及生命周期、知识长文与搜索、深度学习、方法与实践、时间/活动直接操作、完成/中断/回归、今日整理长文、正式原则、候选确认、因果轨迹和历史。
- **由 Agent 主导、前端确认**：深层理解、未知问题、方法建议、行动建议、长期方向和阶段目标建立。
- **转入后台**：行动阻力、长期模式、趋势和节奏推断、正常的每日整理生成；只有它们改变认知、学习、方法或实践时才把必要结果交给用户。
- **合并重组**：今日学习归认识世界，行动与活动归改造世界，即时反馈归现实课题，已验证更新归长期资产。
- **删除重复展示**：知识、复盘和原则的静态演示样例，以及与现实课题无关的固定打卡主线。
- **新增统一交互**：全局快速记录入口，允许长文、批量或结构化输入先保存原始内容，再由系统建议关联，不要求用户预先分类。

完整逐项去向以 `SSD/05-frontend-design-direction.md` 为准。功能审计已经完成，并在下方 V4.3 收口为最终目标导航；页面代码尚未修改。

## V4.3 最终前端信息架构（已批准目标）

最终用户导航收束为「现实 / 认识 / 实践 / 积累」，并提供贯穿所有页面的「记一笔」。

- **现实**：默认首页，展示一个现实问题调用了哪些私人知识、怎样形成针对性判断与方法、当前缺口和准备怎样验证。
- **认识**：比较被调用知识分别解释什么、怎样组合、适用边界和冲突，并形成有来源的方法候选。
- **实践**：把经确认的方法放进真实条件中检验，明确触发条件、步骤、观察指标、反例和结果记录。
- **积累**：展示现实结果怎样支持、修正或淘汰知识与方法，保留来源链、适用条件、失效条件和个人方法历史。
- **记一笔**：现实记录、新知识和实践结果先保留原文；新知识还需提取问题、方法、条件、边界与可验证动作，经确认后才参与现实分析。

`/reflect` 与 `/way` 不再作为最终一级导航；原活动与反馈进入实践或现实，今日整理、方向、原则与候选进入积累，模式/趋势/节奏推断进入后台。旧链接在新主路径验收前继续兼容，正式数据不删除。

## 0. 能力模型

| 能力 | 用户最终得到什么 | 现有实现归属 |
|------|------------------|--------------|
| 人生方向 | 多个成长领域、阶段目标和当前实践的长期地图 | /way + goals/wisdom，待补层级关系 |
| 现实输入 | 真实知识、行动、活动、状态和中断记录 | /know、/act、/reflect |
| 当前选择 | 当下最值得做且属于用户可控范围的最小实践 | /act 主行动、schedule suggest |
| 实践与回归 | 执行、降档、中断、重新开始和证据记录 | /act、activities、schedule 状态 |
| 反馈与整理 | 从事实和反馈生成下一次调整 | /reflect、daily-review |
| 认知成长 | 认知候选、来源证据、用户确认后的正式原则 | /way、cognitive asset candidate、wisdom |
| 节奏调整 | 多方向之间的动态焦点和密度建议 | daily-plan、pattern/resistance/trend |
| Agent 入口 | 用自然语言完成上述能力的日常编排 | Hermes 结衣 + 产品 adapter |

## 0.1 统一判断

- “任务”只是当前实践的一个实现形态；没有成长领域、阶段目标或反馈关系的任务不能成为结衣的核心成果。
- “知识”只是现实输入的一种来源；没有进入问题、实践或认知成长链路时，不作为核心进展。
- “陪伴”和“轻量整理”是对话方式，不是产品顶层模块；它们最终都可以回到当前选择或明确停留在陪伴状态。
- Agent 的建议不等于用户确认；降档、暂停、改方向和原则提升都必须保留确认边界。

## 0.2 第一条真实主路径

```text
成长领域 → 阶段目标 → 当前实践 → 完成 / 中断 → 回归 → 反馈
```

- 成长领域和阶段目标由用户确认后保存，可长期并行存在。
- 当前实践必须挂到一个阶段目标；普通旧行动继续兼容，但不冒充已挂接实践。
- 中断不清零；回归恢复同一实践并保留事件轨迹。
- 完成、中断和回归都是反馈，不自动改变长期方向。

## P1 功能

- 遗忘曲线复习卡片：在 /act 展示 `daily_plan.review`。
- 长期目标查看：在 /way 展示 goals；/dao 仅为兼容 alias/历史称呼。
- 智慧卡片：在 /way 展示 wisdom；底部导航仍使用 /way。
- 反复模式识别：后台分析长期数据。
- Hermes 接管每日整理：从手动触发逐步过渡到 cron 自动生成。
