## P0-1：知识输入与转行动

**页面：/know**

### 描述

用户可以在知页导入外部知识，将知识保存后在列表查看，并可拆解为具体行动进入行页执行队列。

### AC

| ACID | 验收条件 | 优先级 |
|------|----------|--------|
| AC-1.1 | 用户在知页看到知识导入输入框，可以粘贴外部内容 | P0 |
| AC-1.2 | 保存后知识出现在知页知识列表 | P0 |
| AC-1.3 | 知识列表展示内容摘要、时间、来源、标签 | P0 |
| AC-1.4 | 每条知识带有“拆解 → 行”按钮 | P0 |
| AC-1.5 | 拆解后可创建任务，并出现在 /act 执行队列 | P0 |
| AC-1.6 | API 不可用时显示“接口未就绪”，不伪装成功 | P0 |

## P0-4：活动记录、今日整理与节奏建议

**页面：/reflect**

### 描述

思页记录真实活动，并只保留一个“今日复盘”输入窗口。用户只打一段自然语言，前台不再要求区分今日备注、明日调整、心情或思想；后台先完整保存原文，今日整理读取这段原文与活动/行动数据，连续沉淀一段时间后再做长期模式分析。

### AC

| ACID | 验收条件 | 优先级 |
|------|----------|--------|
| AC-4.1 | 思页有活动记录输入区域 | P0 |
| AC-4.2 | 思页只有一个“今日复盘”输入区域，用于保存整段原文 | P0 |
| AC-4.3 | 保存后当天活动出现在活动列表 | P0 |
| AC-4.4 | 有“生成今日整理”触发按钮 | P0 |
| AC-4.5 | 今日整理读取活动、行动完成情况与统一复盘原文，生成今日总结、节奏风险、后续调整建议、认知资产候选 | P0 |
| AC-4.6 | 节奏建议展示偏执行 / 偏整理 / 偏恢复 | P0 |
| AC-4.7 | API 不可用时显示失败，不使用假数据 | P0 |
| AC-4.8 | 复盘输入不自动生成日程；行动/日程仍只归 /act 链路处理 | P0 |

## P1 功能

- 遗忘曲线复习卡片：在 /act 展示 `daily_plan.review`。
- 长期目标查看：在 /way 展示 goals；/dao 仅为兼容 alias/历史称呼。
- 智慧卡片：在 /way 展示 wisdom；底部导航仍使用 /way。
- 反复模式识别：后台分析长期数据。
- Hermes 接管每日整理：从手动触发逐步过渡到 cron 自动生成。
