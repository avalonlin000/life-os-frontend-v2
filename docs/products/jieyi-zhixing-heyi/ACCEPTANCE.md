## V4 整体重做验收（最高优先级）

| # | 验收点 | 通过条件 |
|---|---|---|
| V4-1 | 当前课题入口 | `/`、`/jieyi/`、`/reality` 进入同一当前课题体验；无课题时可自然语言建立 |
| V4-1A | 方法—实践因果关系 | 实践必须绑定同课题已确认方法；后续反馈可追溯当时采用的方法 |
| V4-1B | 反馈自动更新候选 | 每次反馈自动形成世界观与方法论两类待确认候选，并保留反馈、实践和原判断来源 |
| V4-1C | 正式态诚实显示 | 首页只把 confirmed 认识/方法显示为当前结论；candidate/rejected 不冒充正式内容 |
| V4-2 | 认识聚合 | 同一课题能区分事实、知识、候选理解和待回答问题，并保留来源/状态 |
| V4-3 | 改造聚合 | 方法、时间实践、完成/中断/回归都关联同一课题；旧行动仍可用 |
| V4-4 | 反馈更新 | 结果能追溯到原认识和方法，并分别产生世界观/方法论更新候选 |
| V4-5 | 用户确认 | 课题建立、焦点切换、正式认识、方法、重要实践和写入都需明确确认 |
| V4-6 | Agent 交接 | 新会话能读取课题、提出建议、确认后写入并读回；无工程工具越权 |
| V4-7 | 独立维护 | Hermes 能用固定入口完成健康检查、备份、兼容迁移和回归，不依赖人工读代码 |
| V4-8 | 真实改善 | 同一课题连续运行后出现可感知现实变化和至少一次认识/方法更新 |
| V4-9 | 单步式前端 | 手机首屏只突出当前课题、当前阶段和唯一下一步；认识/实践/反馈均可切换但不同时铺满 |
| V4-10 | 反馈不串线 | 多项实践存在时，反馈由用户明确选择对应实践，并显示方法→实践→反馈→两类更新关系 |
| V4-11 | 诚实状态 | 焦点读取失败不进入新建页；知识读取失败不冒充空知识；正式库不写入验收样例 |
| V4-12 | 课题生命周期 | 编辑可读回；暂停/完成二次确认；无焦点进入课题库；暂停课题可显式继续，已完成课题保留历史 |
| V4-13 | 上下文连续 | 实践事件可展开；从知/行/思/道返回现实课题后仍保持离开前阶段 |
| V4-14 | 长期观察边界（Planned） | 单次信号不直接生成心理/人格结论；工作假设保留依据、不确定性和可纠正性；只有经现实验证与用户确认才成为正式认识或方法 |
| V4-15 | Agent 自然协作（Planned） | 日常对话不退化为逐字段填表；Agent 能结合长期证据主动指出矛盾、解释依据并提出一次现实验证，同时不声称掌握“真正状态” |
| V4-16 | 前端工作台（Planned） | 前端稳定承载认知长文、学习材料、方法步骤、时间实践、快速/批量记录、结果验证和纠错确认；不展示人格评价，也不强迫用户先完成系统分类 |
| V4-17 | 四功能面导航（Planned） | 手机和桌面均以现实/认识/实践/积累为同一导航顺序；现实为默认入口，旧 reflect/way 链接仍可达但不再出现在一级导航 |
| V4-18 | 全局快速记录（Planned） | 任意页面可打开“记一笔”，不分类也能保存原文；可选快捷动作不会阻止普通输入；保存后可查看并纠正系统建议的关联 |
| V4-19 | 旧入口无损迁移（Planned） | reflect 的活动/反馈/整理和 way 的方向/原则/候选在新位置可读回；现有正式数据不删除，旧链接在迁移期继续兼容 |
| V4-20 | 固定知识—现实转化 M1 | 现实课题只调用真实私人知识并展示相关性、适用条件、边界与缺口；无知识时诚实保留缺口；方法经确认后绑定实践；反馈只更新实际采用的方法；个人方法版本保留知识、实践和反馈来源 |

下方 P0-P4 只作为旧入口兼容验收，不得替代 V4 主路径。V4-14 至 V4-19 是 2026-07-22 批准的下一阶段产品契约，未完成真实 Agent 与前端验收前不得标记 Verified。
