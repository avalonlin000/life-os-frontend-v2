## 2. 产品边界

结衣是长期只服务钧钧的私人人生系统。2026-07-21 起，唯一最高产品闭环为：

```text
人生方向选择值得改变的现实课题
  → 认识世界：看清事实、矛盾、客观条件和已有知识
  → 改造世界：形成方法，组织时间与行动，进入现实实践
  → 实践反馈：记录结果、阻力、新事实和新问题
  → 更新世界观与方法论，继续下一轮实践
```

结衣项目不拥有：

- 电竞、LOL、队伍、TK、TS、盘口和日报工作台。
- 普通 todo 清单；行动必须服务于反馈调整。
- 心理诊断、重问卷或假数据填充。

结衣可以读取外部上下文来辅助反馈，但外部上下文不是结衣核心数据正源；交易关联目前只应作为可选只读上下文，不能让结衣主链路依赖小雪数据。

### 2.1 已批准的产品能力总图

```text
钧钧：人生方向、价值判断、最终确认
                  ↓
现实课题：当前不满、事实、矛盾、客观条件
                  ↓
认识世界：现实记录＋人类知识 → 当前世界观与判断
                  ↓
改造世界：方法论 → 时间、行动与现实实践
                  ↓
现实结果：变化、阻力、新事实和新问题
                  ↓
反馈更新：修正世界观与方法论，继续下一轮实践
                  ↺
```

现有 `/know`、`/act`、`/reflect`、`/way` 只是历史入口；成长领域、阶段目标、当前实践和回归能力只是可复用的下层方法，不能继续代替现实课题成为产品组织中心。

### 2.3 前端功能去向审计（2026-07-22）

- 保留页面优势：现实课题管理、长知识阅读、深度学习、方法与实践操作、活动与长文记录、结果验证、原则与候选确认、搜索和历史。
- 收回 Agent：深层理解、未知问题、方法/行动建议、方向和阶段目标建立；前端负责展示、纠错和确认。
- 转入后台：阻力、模式、趋势、节奏推断和正常每日整理生成；只有改变认知、学习、方法或实践时才展示必要结果。

- 功能审计先收出当前现实课题、认识世界、改造世界、长期资产和跨页面全局快速记录，随后已批准对应最终导航。

2026-07-22 后续确认：最终目标导航已定为「现实 / 认识 / 实践 / 积累」＋全局「记一笔」。当前代码仍运行现实/知/行/思/道兼容入口，因此该总图属于已批准设计、尚未实现状态。

### 2.4 固定知识—现实转化方法重新开工审计（2026-07-22）

已锁定的目标方法为：

```text
现实记录
  → 调用私人知识
  → 比较适配、边界、冲突和知识缺口
  → 形成并确认方法
  → 实践检验
  → 更新个人系统
```

当前真实产品只闭合了缩短链：

```text
现实记录
  → 用户手动写认识和方法
  → 用户确认
  → 建立实践并记录反馈
  → 确认课题内部更新候选
```

| 固定环节 | 当前事实 | 结论 |
|---|---|---|
| 现实记录 | 课题建立、编辑、切换、暂停、完成、恢复和时间线可用 | 可直接复用 |
| 调用私人知识 | 只能手动搜索并关联；主流程允许有事实后跳过知识 | 固定方法未成立 |
| 适配、边界、冲突和缺口 | 深度学习有边界提示，但不绑定现实课题；没有正式比较或缺口结构 | 核心缺失 |
| 方法形成与确认 | 候选确认、同课题约束和方法绑定实践可用 | 骨架可复用；缺知识来源链 |
| 实践与反馈 | 完成、中断、回归、指定实践反馈和事件轨迹可用 | 骨架较完整 |
| 更新个人系统 | 反馈只形成课题内认识/方法候选；确认后不进入长期私人知识或个人方法 | 核心缺失 |

正式数据现状：

- `knowledge` 共 12 条，其中 8 条为 `xiaobai-smoke` 工程验收内容；非 smoke 真实内容 4 条，`is_core` 为 0。
- 正式现实课题和课题条目均为 0；尚无一条真实“知识 → 课题 → 方法 → 实践 → 反馈”证据。
- 52 条行动中 22 条来自知识拆解，且全部来自 smoke 知识；没有行动关联现实课题或已确认方法。
- 原则接口显示的“已验证”内容均无关联实践，当前只能证明曾被确认或沉淀，不能证明经过现实检验。

已发现的正确性问题：反馈生成方法更新时使用课题最新已确认方法，而不是该实践实际绑定的 `method_entry_id`；多方法课题可能更新错对象。进入真实使用前必须修复并建立回归保护。

项目诊断：现有现实课题、知识读写、候选确认、方法绑定实践、实践事件、反馈候选和原则提升接口可以复用；需要新增和重接的是“课题驱动知识调用—比较与缺口—来源化方法—实践证据—个人方法版本”关系契约。当前不应先扩大全库语义搜索或大改四页。

## 7. 已知冲突与风险

### 7.1 DailyPlan 双源导致已确认断链

当前存在两份计划文件/数据路径：

- `/home/ubuntu/workspace/knowledge/daily/daily_plan.json`：当前有 2026-07-18 内容。
- `/home/ubuntu/workspace/knowledge/wiki/小雪电竞/原始资料/daily-data/daily_plan.json`：当前文件日期为 2026-07-16。

此前 `/api/daily-plan` 读取第二个路径，并且只在日期完全匹配时返回内容；2026-07-18 请求实际返回空的 `learn/review/doTasks`。当前 GET 已改为先读产品数据库，再从 `/home/ubuntu/workspace/knowledge/daily/daily_plan.json` 做兼容读取，并在响应中标记 `source=legacy_file`；同时提供显式 `POST /api/daily-plan/refresh` 导入入口。产品数据库 `daily_plan` 表最新记录目前为 2026-06-29，也没有覆盖当前计划。

影响：行页的今日学习、复习和计划行动此前为空；计划写入、刷新和重新进入仍未完全形成单一读回链路。

建议：以 `daily_plan` 表或产品-owned plan adapter 作为唯一产品正源；旧文件仅做兼容导入和回退读取，迁移完成前不得删除。

### 7.2 测试数据污染知识材料

`knowledge` 表当前仍有 8 条标题包含 `xiaobai-smoke` 的验证材料。`/api/jieyi/today/aggregate` 的材料列表会返回其中部分记录；行页和模式检测虽有工程噪声过滤，但知页材料入口并未全局过滤。

影响：用户可能看到工程验证记录，被误认为真实知识或输入信号。

建议：测试写入使用隔离数据/标记，产品查询在服务层统一排除测试记录；历史测试记录先盘点、备份、确认后再清理。

### 7.3 交易上下文边界不清

`mood` 有 `trade_ids` 字段，今日整理实现也存在交易关联读取；但产品边界明确结衣不负责电竞交易分析。交易数据只能作为可选只读上下文，不能成为知识、行动、复盘主链路的必需依赖。

影响：小雪数据服务或交易表异常时，结衣每日整理可能被连带拖垮；产品语义也会与小雪边界混淆。

建议：以可选外部上下文适配器隔离，缺失时仍生成不含交易段落的结衣整理。

### 7.4 业务聚合重复在浏览器

模式、阻力和趋势目前由前端逐日调用多个 API 后计算。网络慢、部分接口失败或日期时区不一致时，三个页面可能得到不同窗口结果。

影响：请求数量高，失败边界不清，页面逻辑重复；共享 service 变更还可能影响小雪。

建议：逐步移到产品 API 的只读派生端点，前端仅消费稳定结构；保留旧 service 兼容出口。

### 7.5 路由/服务分层正在收口

知、行、思主链、原则、首页聚合、深度学习、每日整理、行动建议、知识拆行动和长期复盘 AI 已进入 `app/services/jieyi/` 领域服务；Agent 只剩少量历史兼容读写入口。

当前主入口和 Agent 受限产品适配器已按产品域拆分。适配器已经实现并通过合同测试，但尚未挂载到真实 jieyi profile，真实飞书新会话读写仍待一次授权后验收。

下一步：继续收口目标/笔记等非主链兼容入口、Cron 写入和旧 fallback；路由保持参数校验、调用和错误映射，对外 API 形状保持兼容。
