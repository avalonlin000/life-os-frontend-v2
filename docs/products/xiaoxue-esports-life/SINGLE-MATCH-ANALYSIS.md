## 3. 输入信息顺序

小雪拿到信息时，按“够用优先”而不是追求全量：

1. 阵容确认：蓝红方、五个英雄、位置、是否已最终锁定。
2. 队伍基本面：双方队伍、赛区、近期状态、风格、三维/画像。
3. TS / 基本面：如是 MSI 或已有底表，引用 mu / sigma / TS / risk_gap；没有就明确缺口。
4. 版本与英雄语境：版本优先级、英雄强势期、阵容组合常见打法。
5. 盘口/交易语境：只作为风险和市场预期背景，不替钧钧下交易结论。

## 4. 输出结构

每次单场 / BP 分析默认输出以下结构，长度按钧钧问题压缩：

1. 阵容确认
   - 先复述双方阵容和位置，避免在错误阵容上分析。

2. 控制量化
   - 明确开团、反开、点控、AOE 控、视野控制、资源团控制的强弱。
   - 不只说“控制多/控制少”，要说明控制来自哪里、几级后成型、是否稳定。

3. 基本面 / TS
   - 引用队伍基本面、三维、画像、mu / sigma / TS / risk_gap 等可用底表。
   - TS 是保守实力下界，不是绝对实力；sigma 用于提示稳定性和爆冷空间。

4. 线权时间边界
   - 拆 1-6 级、第一条小龙/巢虫、14 分钟前后、一二件套、三件套后的主动权变化。
   - 说清楚谁先有线权、谁必须避战、谁拖到哪个时间点会舒服。

5. 核心链
   - 用一句话说阵容怎么赢：例如“上中野先手链 → 控资源 → 双 C 收割”。
   - 同时说明这条链的前置条件和断点。

6. 24场景定位
   - 把本场放进 24场景矩阵：强开、反开、poke、前中期滚雪球、后期大核、边线牵扯、资源团阵地等。
   - 不是硬背分类，而是帮助钧钧快速知道“这把按什么剧本看”。

7. 风险点
   - 执行难度、缺伤害、缺开团、缺前排、线权断档、容错低、视野压力、被反开、被拖后期等。
   - 明确哪些风险是阵容结构风险，哪些是队伍基本面/状态风险。

8. 交易边界
   - 可以提示“市场可能低估/高估的点”“赔率和基本面的分歧”。
   - 不替钧钧下结论，不直接给买卖指令，不把阵容分析包装成交易建议。

