# 小雪电竞人生 — LOL 横向基本面前端方向

> 状态：讨论收口稿
> 主责：小白
> 原则：日报负责每日比赛；前端负责横向、系统化、可长期查看的 LOL 基本面知识。

---

## 1. 核心结论

小雪前端现阶段不应该主打“每天比赛”。

每天比赛、赛程重点、当日观察，已经由小雪日报承接。前端要做的是另一个角色：

> 把 LOL 的大基本面、队伍横向知识、TK / 概念图 / 分类图系统化地摆出来，成为钧钧平时查基本面、对比队伍、整理思路的桌面工作台。

盘口/交易类内容不并入基本面主页面，而是另起独立「交易页」。分析、盘口、赔率、临场想法都放在交易页，避免和基本面混在一起导致后续逻辑自动生成得乱。

---

## 2. 前端、日报、赛程数据的分工

### 日报负责

- 每天有哪些比赛。
- 当天重点赛程。
- 今日关注点。
- 当天临时信息、动态、结果。
- 钧钧日常问小雪时的短平快查询。

### 赛程数据负责

- 赛程不是给钧钧在前端“看比赛表”的主功能。
- 钧钧看赛程会用别的 App。
- 小雪保留赛程数据，是为了导入、关联、分析比赛时能识别赛事、队伍、阶段和对局背景。
- 前端只在需要分析某个赛事/队伍/对局时引用赛程上下文，不把赛程卡片放主位。

### 前端负责

- 横向查看 LOL 基本面。
- 系统化整理队伍、版本、风格、TK、概念图。
- 罗列每个队伍的关键基本面。
- 给钧钧一个长期可回看的知识工作台。
- 当前重点研究 MSI 这类国际赛事时，把外卡/外赛区队伍纳入同一套横向基本面框架。
- 给盘口思考留输入窗口，但不替代钧钧自己的分析。

一句话：

```text
日报 = 每天看什么
赛程数据 = 给分析做关联，不给用户看表
前端 = 平时怎么看懂这些队伍、版本和国际赛环境
```

---

## 3. 前端主内容：横向基本面

前端主页面不按“今天某场比赛”组织，而按横向知识组织。

建议主结构：

```text
LOL 横向基本面
├─ 版本 / 体系
├─ 赛事环境：常规 LPL/LCK 与 MSI 国际赛
├─ 队伍横向表
├─ 队伍画像
├─ 选手画像
├─ TK / 战术概念
└─ 概念图 / 分类图
```

### 3.0 赛事环境：常规联赛 vs 国际赛

小雪平时主要服务 LPL / LCK 的长期基本面查看；当前阶段如果重点研究 MSI，则要把国际赛事环境单独纳入基本面，而不是做成赛程表。

国际赛 / MSI 的特点：

- 队伍池混杂：LPL、LCK、外赛区、外卡队同时出现。
- 外卡和外赛区资料稀疏，不能按 LPL/LCK 的完整画像要求硬填。
- 判断重点从“熟悉队伍内部细节”转向“赛区差异、版本适配、BO 经验、强弱断层、信息不足风险”。
- 前端需要把外卡队伍放进横向表里，但明确标注资料可信度和缺口。
- 赛程只作为识别赛事和队伍池的背景数据，不作为用户主要查看对象。

因此前端需要有一个“赛事环境”视角：

```text
常规 LPL/LCK：看长期队伍风格、选手、版本适配
MSI：看跨赛区强弱、外卡未知量、版本理解差、BO 稳定性
```

### 3.1 版本 / 体系

展示当前或近期版本的大方向：

- 版本风格
- 强势体系
- BP 优先级
- 英雄 / 位置 / 资源优先级
- 版本对不同队伍风格的影响
- 来源：用户记录、B站、微信、TK、日报沉淀

### 3.2 队伍横向表

这是前端的核心之一。

不是只点开一个队，而是横向罗列每个队伍的关键基本面：

- 队伍风格
- 强项
- 弱点
- 稳定性
- 版本适配
- 关键选手
- 风险点
- 相关 TK / 画像链接

目标是让钧钧快速横向比较：

```text
哪些队伍吃版本
哪些队伍只是名气大
哪些队伍近期可能有趋势
哪些队伍风险很高
```

### 3.3 队伍画像

保留现有队伍画像和三维数据，但它不再是唯一入口。

用途：

- 点某个队时查看纵向详情。
- 和横向表互相跳转。
- 展示队伍长期风格和结构性问题。

### 3.4 选手画像

作为队伍基本面的补充。

用途：

- 看关键位。
- 看英雄池、状态、风险点。
- 解释为什么某队风格能成立或失效。

### 3.5 TK / 战术概念

继续保留 TK 搜索和编辑，但更偏“依据库”。

用途：

- 查版本概念。
- 查战术体系。
- 查某队/某选手相关材料。
- 作为横向表和画像的依据来源。

### 3.6 概念图 / 分类图

这是前端应该重点保留的横向查看能力。

用途：

- 看 TK 之间的关系。
- 看队伍与战术概念的关系。
- 看版本、体系、队伍风格之间的关联。

## 4. 交易页独立承载分析

交易分析、盘口、赔率、临场观察不放进基本面主页面。

前端顶部保持两个清晰入口：

```text
基本面 ｜ 交易
```

交易页只做钧钧自己的分析空间：

- 钧钧手动输入当前想法。
- 钧钧手动输入盘口、赔率、临场观察。
- 可以手动引用基本面里的队伍、TK、版本依据。
- 「带入当前队伍」只带入对象和对局背景，不自动生成方向、不自动生成后续判断链。
- 不强制保存。
- 不强制结算。
- 不把命中率、输赢统计作为主展示。

这页的定位是：

> 基本面页只放客观资料和横向比较；交易页单独放分析草稿。小雪不替钧钧自动续写复杂交易逻辑，避免越生成越乱。

## 5. 来源处理原则

来源继续统一。

来源包括：

- 钧钧自己记录
- B站导入
- 微信导入
- TK 手动条目
- 日报沉淀
- 队伍画像 Wiki
- 选手画像 Wiki
- SQLite 中的赛程、队伍、三维数据

前端不按来源拆页面，而是按用途展示：

```text
来源用于追溯
基本面用于展示客观资料
概念图用于横向关系
交易页用于钧钧自己单独写分析、盘口、赔率和临场想法
```

## 6. 当前页面改造口径

现有模块对应关系：

| 现有内容 | 新定位 |
|---|---|
| MSI / 每日赛程 | 不作为看赛程入口；作为当前国际赛事研究的队伍池、赛事阶段和对局背景数据 |
| LPL / LCK 队伍 | 平时横向基本面的主对象 |
| INTL / 外卡队伍 | 国际赛横向表补充对象，允许资料稀疏并标注缺口 |
| 三维数据 | 队伍横向表 + 队伍详情 |
| 队伍画像 | 队伍详情 / 横向表依据 |
| 选手信息 | 队伍详情的补充维度 |
| TK 搜索 | 基本面依据库 |
| TK 概念图 | 横向知识图谱 |
| 交易记录 | 从基本面主流程移出，改为独立交易页的分析草稿 |
| 统计、结算、命中率 | 不放主流程 |

---

## 7. 一句话产品口径

> 小雪前端不是“每日比赛页”，每日比赛由日报和外部 App 承接；前端是 LOL 横向基本面工作台：平时系统化展示 LPL/LCK 的版本、队伍、选手、TK、概念图和分类图；研究 MSI 时，把外赛区/外卡队伍放入同一套横向基本面框架，突出赛区差异、资料缺口和国际赛环境；交易另起独立页面，只留给钧钧自己写分析、盘口、赔率和临场想法，不和基本面混放。
