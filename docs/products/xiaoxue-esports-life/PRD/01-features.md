## P0-1：基本面 / 交易页签

**入口：顶部栏**

### 描述

前端一级结构改为「基本面 / 交易」。默认进入基本面；交易页只提供钧钧自己手动写分析、盘口、赔率、临场想法和可选保存，不和基本面混放。

### AC

| ACID | 验收条件 | 优先级 |
|------|----------|--------|
| AC-1.1 | 顶部展示「基本面 / 交易」页签 | P0 |
| AC-1.2 | 默认进入「基本面」 | P0 |
| AC-1.3 | 切到「交易」时隐藏基本面主卡，展示交易分析窗口 | P0 |
| AC-1.4 | 交易统计、命中率、结算不出现在主流程 | P0 |

## P0-2：LOL 队伍横向表

**入口：基本面页**

### 描述

横向罗列 LPL / LCK / INTL 队伍的关键基本面：赛区、三维摘要、版本/风格摘要、资料状态、TK/画像存在性。

### AC

| ACID | 验收条件 | 优先级 |
|------|----------|--------|
| AC-2.1 | 调用 `/api/fundamentals/teams?scope=msi|lpl|lck|intl|all` | P0 |
| AC-2.2 | 表格展示队伍、赛区、三维、摘要、资料状态 | P0 |
| AC-2.3 | 外卡/资料不足队伍明确标注“资料不足” | P0 |
| AC-2.4 | 点击队伍能加载队伍画像、三维、TK | P0 |
| AC-2.5 | 不用前端逐队 N 次请求拼表 | P0 |

---

## P0-3：MSI 国际赛环境

**入口：基本面页 / MSI scope**

### 描述

MSI 不作为赛程表，而作为国际赛环境研究：队伍池、赛区分布、外卡未知量、资料缺口、版本适配和 BO 稳定性。

### AC

| ACID | 验收条件 | 优先级 |
|------|----------|--------|
| AC-3.1 | 调用 `/api/fundamentals/msi` 返回 event、teams、regions、missing_profiles、missing_3d | P0 |
| AC-3.2 | 页面展示 MSI 队伍池数量、资料缺口、赛区分布 | P0 |
| AC-3.3 | 赛程数据仅作为队伍池和对局背景，不展示成主赛程表 | P0 |
| AC-3.4 | INTL / 外卡队伍允许资料稀疏但必须标注 | P0 |

## P0-7：盘口分析窗口

**入口：盘口页**

### 描述

盘口页只给钧钧自己写盘口、赔率、临场想法，可引用基本面依据；不做强制记录、结算、命中率主面板。

### AC

| ACID | 验收条件 | 优先级 |
|------|----------|--------|
| AC-7.1 | 可输入比赛/盘口/方向/理由 | P0 |
| AC-7.2 | 可引用当前队伍或基本面上下文 | P0 |
| AC-7.3 | 可选保存，当前可复用 `/api/trades` | P0 |
| AC-7.4 | 不强制结算，不把命中率放主展示 | P0 |

## P1 功能

### P1-1：电竞日报

定时整合赛程、结果、队伍动态、Wiki 更新，生成每日电竞日报。日报负责“每天看什么”，前端不重复做赛程 App。

### P1-2：双分析师视角

提供中年电竞人、心语悦无言两套分析视角，用于队伍详情交叉验证。

### P1-3：盘口草稿独立表

如果后续不想复用 `/api/trades`，新增 `market_notes` / `/api/market-notes`，让盘口语义和历史交易记录分离。
