## 1. 项目一句话定义

> 小雪电竞人生是面向钧钧的电竞基本面工作台——平时系统化查看 LPL/LCK 队伍、选手、版本、TK 与概念图；研究 MSI 时把外赛区/外卡队伍纳入同一套横向基本面框架；盘口另起分页，只给钧钧自己做分析和可选保存。

---

## 2. 系统目标

| 序号 | 目标 | 说明 |
|------|------|------|
| T1 | 横向基本面 | 横向查看版本/体系、队伍风格、三维、画像、选手和 TK，而不是只点开单队。 |
| T2 | MSI 国际赛环境 | 研究 MSI 队伍池、跨赛区强弱、外卡未知量、版本适配、BO 稳定性和资料缺口。 |
| T3 | 知识依据 | 将 B站、微信、用户记录、TK、日报沉淀、队伍/选手画像统一作为基本面依据。 |
| T4 | 独立交易页 | 交易页只提供手动分析、盘口、赔率、临场想法和可选保存；不和基本面混放，不自动生成复杂后续逻辑。 |
| T5 | 数据可信 | 真实数据库 / Wiki / API 来源优先；资料不足必须显式标注，不伪装完整。 |

## 3. 用户场景

### 核心场景：平时看基本面

用户打开小雪电竞人生 → 默认进入「基本面」 → 查看版本/体系、队伍横向表、队伍画像、选手画像、TK 和概念图 → 自己形成对队伍和版本的理解。

### 核心场景：MSI 国际赛研究

用户进入基本面页 → 切到 MSI 视角 → 查看 LPL/LCK/INTL 队伍池、资料缺口、跨赛区差异、外卡未知量、BO 稳定性 → 决定哪些队伍需要补资料或重点关注。

### 补充场景：盘口分析

用户进入「盘口」分页 → 输入盘口、赔率、临场想法 → 可引用当前基本面依据 → 可选保存为草稿或后续沉淀；小雪不强制结算、不显示命中率作为主功能。

### 补充场景：日报与赛程

每天比赛、赛程重点、当日观察由日报和外部 App 承接。小雪保留赛程数据，是为了关联赛事、队伍、阶段和对局背景，不作为前端看赛程表的主功能。

## 4. 产品边界

| # | 不做什么 | 理由 |
|---|----------|------|
| B1 | 不做赛程 App | 钧钧看赛程用别的 App；小雪赛程数据只服务分析关联。 |
| B2 | 不做交易填空题 | 盘口判断由钧钧自己完成，小雪只提供输入窗口和基本面依据。 |
| B3 | 不把命中率/结算放主流程 | 当前重点是基本面，不是交易记录系统。 |
| B4 | 不做移动端优先 | 小雪工作台是桌面端分析工具，1920px 信息密度优先。 |
| B5 | 不伪装实时数据 | 队伍、赛程、画像、TK 不可用时必须显式失败或标注资料不足。 |
| B6 | 不和结衣混成一个 Life OS 页面 | 小雪是电竞人生产品，结衣是知行合一产品。 |
