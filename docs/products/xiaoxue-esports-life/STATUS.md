## 总结

当前每日日报自动运行由钧钧主动暂停，原因是仍有日报模块待完善。暂停属于用户决定，不是系统故障；在剩余模块修好并真实验收前不恢复自动发布。

当前结论：定位、职责、窗口边界、完整主流程、最终方案和落地顺序已经确认；底层模块化、三层架构、接口隔离与单模块故障隔离已经落地。产品体验仍在按最终方案迁移，不能把“接口存在”或“文档已写”视为完整主流程已经走通。

## 当前有效系统定义

> 小雪是钧钧长期共同发展的个人电竞交易助理，目前主业是 LOL。产品采用一个主体、三个窗口、一套后台。

三轴模型：

- 项目轴：LOL / KPL / DOTA2 / CS2 / Valorant / 其他。
- 使用时间轴：交易前 / 交易时 / 交易后。
- 产物对象轴：给钧钧看的 / 给模型用的 / 给系统审计的。

八层能力：

```text
使用入口、长期依据、知识生产与沉淀、单场判断、阵容变量、市场对照、复盘校准、数据工程与自动化
```

## 当前主链路边界

| 事项 | 当前口径 |
|---|---|
| 日报 | 每日唯一默认入口；展示已加工成品，不分析未出现 BP |
| 工作台 | 只保留队伍资料 / 当前赛事 / TK资料库 |
| 小雪对话 | 动态查询、预案、阵容判断、三层对照和复盘入口 |
| 日报生产 | `daily_pipeline.py --publish` 是唯一入口，负责冻结、渲染、校验、发布和回读 |
| 日报内容 | 昨日校准 → 当前大周期预案摘要 → 今日赛程 → 通用版本 → 逐场卡 → 近 7 天正式 TK → 缺失提示；当前保持用户暂停 |
| 豆包搜索 | 只服务赛前 `public_opinion`；默认一次统一问、每日实际请求最多 3 次，非舆论不得使用 |
| 飞书发布 | 主文档作固定导航入口，正文和 TK 按块上限自动分卷并逐卷回读 |
| 临场记录 | `/api/market-notes` 是当前真实记录主链，但作为隐藏辅助，不进主导航 |
| 旧交易记录 | `/api/trades` 只兼容，不回主流程 |
| 队伍交易备注 | 跟随队伍 TK，`type=trading_note`，不新增交易 TK 实体 |
| 纯十英雄阵容判断 | `lol-lineup-analysis` v7.3；默认等经济、正常发育、同等操作水平，给总体与四阶段强度、比赛剧本、比赛画像和观赛信号 |
| 交易判断 | 明确要求“我的交易系统 / 交易理念 / 赛前预案”时进入 `junjun-trading-system`，另行核验队伍、市场、趋势、分歧和风险 |
| 旧 BP 结构契约 | `/api/lineup-workflow/prepare` 仍是历史结构化草稿接口，不代表当前纯阵容方法，也不得作为当前对话路由正源 |
| 赛后复盘 | 回到小雪对话，由小雪读取 market_notes 后讨论；后台写回仍须 preview + 明确确认，不自动写知识 |
| 数据就绪 | 05:30 合并 ScoreGG+TS 并生成 readiness manifest；日报发布前强制门禁 |
| 自动交易 | 禁止；小雪只做证据、分歧、风险和复盘校准 |
| 知识长流程 | 完整阶段整理继续统一编排；独立请求不会顺带运行后续步骤，点名选手时只允许修改该选手画像 |

## 2026-07-13 真实使用修正 — Verified

- 今日赛程为空时，工作台明确显示“今日无比赛”；不生成日报被视为正常休赛状态。
