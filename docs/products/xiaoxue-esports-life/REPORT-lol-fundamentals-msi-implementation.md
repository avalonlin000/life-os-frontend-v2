# 小雪电竞人生 — 横向基本面 + MSI 整合落地可行性报告

> 日期：2026-06-29
> 主责：小白
> 目标：说明小雪前端如何从当前 v1.0 工作台，统一改造成“LOL 横向基本面工作台 + MSI 国际赛研究视角 + 盘口分析分页”。

> 历史报告说明（2026-07-09）：本文保留为 LOL 第一项目长期依据层与旧前端整合方案的历史记录。当前小雪总定义已经由 `PRD/04-trading-methodology-and-taxonomy.md` 接管：小雪是电竞交易辅助系统，当前第一项目是 LOL。文中“LOL 横向基本面工作台”属于长期依据层，不再是系统总定义；旧 `/api/trades` 只保留兼容层含义，当前盘口主链路以 `/api/market-notes` 为准。

---

## 1. 总结论

可行，而且不需要推倒重做。

当前小雪工作台已有的后端、数据源、TK、队伍画像、三维数据、MSI 队伍/赛程接入、概念图入口，都能复用。真正要做的是一次“主次重排 + API 聚合 + 前端结构重组”。

落地口径：

```text
日报 / 外部 App：看每天有什么比赛
小雪后端赛程数据：给分析做关联，不给用户看赛程表
小雪前端主页面：横向查看 LOL 基本面
MSI 视角：当前重点研究国际赛环境、外卡/外赛区队伍、版本适配和资料缺口
盘口分析：另起分页，只给钧钧自己输入和思考，不做交易填空题
```

一句话：

> 小雪前端不是赛程页，也不是交易记录页，而是“LOL 基本面知识工作台”。

## 2. 和当前系统的契合度

### 2.1 已经具备的能力

当前 `/home/ubuntu/xiaoxue-web/` 已有这些能力：

| 已有能力 | 当前实现 | 新定位 |
|---|---|---|
| 队伍列表 | `/api/teams`，含 LPL/LCK/INTL/MSI 队伍 | 队伍横向表入口 |
| 队伍三维 | `/api/team-3d/{team}` + SQLite `team_3d_data` | 横向比较核心字段 |
| 队伍画像 | `/api/wiki/team/{team}` / `/api/profile-full/{team}` | 队伍详情与横向表依据 |
| 选手 | `/api/players?team=` | 队伍详情补充 |
| TK 搜索 | `/api/tk/search`，RAG + 本地文件 | 基本面依据库 |
| TK 新建/删除 | `/api/tk` / `/api/tk/{filename}` | 知识维护保留 |
| 概念图 | `/tk-graph/index.html` | 横向知识图谱主入口 |
| MSI 赛程/队伍 | `/api/schedules?event=MSI`，INTL 队伍兼容 | 不给用户看表，只用于识别队伍池和赛事背景 |
| 交易记录 | `/api/trades` | 移出主流程，改成盘口分页的可选存档能力 |

所以不是缺数据底座，而是当前页面组织还停留在：

```text
队伍画像 + MSI赛程 + 交易记录 + 三维 + TK
```

新的目标是：

```text
版本/体系 + MSI赛事环境 + 队伍横向表 + 队伍/选手详情 + TK/概念图 + 盘口分页
```

### 2.2 当前不契合的地方

当前最不契合的地方有三个：

1. **MSI 被做成“赛程卡片”**
   - 钧钧不会用小雪前端看赛程。
   - 赛程数据应该只作为赛事背景和队伍池识别。

2. **交易记录占了主页面位置**
   - 当前有总记录、已结算、赢输走水、命中率。
   - 这会把产品带偏成交易记录系统。
   - 应该改成独立盘口分页。

3. **缺少横向队伍表**
   - 当前主要是“选一个队看详情”。
   - 钧钧要的是横向看每个队伍的基本面。
   - 需要新增一个聚合接口和前端横向表。

## 3. 统一后的产品结构

建议前端做成两个一级页：

```text
基本面 ｜ 盘口
```

### 3.1 基本面页

基本面页是主页面。

结构：

```text
顶部：小雪工作台 + 搜索/命令栏 + 视角筛选

主区：
1. 版本 / 体系
2. MSI 国际赛环境
3. 队伍横向表
4. 队伍详情抽屉 / 右侧栏
5. TK / 战术概念
6. 概念图 / 分类图入口
```

#### A. 版本 / 体系

展示：

- 当前版本风格
- 强势体系
- BP / 英雄优先级
- 资源优先级
- 版本对队伍风格的影响
- 相关 TK 来源

数据来源：

- `team_3d_data.version_understanding`
- TK 搜索：`版本理解`、`MSI`、`BP`、`体系`
- 小雪电竞 Wiki 战术概念
- B站/微信/手动记录导入的 TK

#### B. MSI 国际赛环境

不是赛程表，而是国际赛研究视角。

展示：

- MSI 队伍池：LPL / LCK / INTL
- 外卡/外赛区资料缺口
- 赛区强弱断层
- BO 经验与稳定性
- 哪些队伍需要补画像 / 补 TK

数据来源：

- `/api/teams` 中 `region=INTL` 或 `league_id LIKE 'MSI%'`
- `/api/schedules?event=MSI` 仅用于识别队伍和阶段
- 已有国际队伍三维/画像，如有则展示；没有则标注“资料不足”

#### C. 队伍横向表

这是本次改造核心。

字段建议：

| 字段 | 来源 | 说明 |
|---|---|---|
| 队伍 | `/api/teams` | short_name / name / region |
| 赛区 | `/api/teams.region` | LPL / LCK / INTL |
| 三维摘要 | `team_3d_data` | 三个维度值 |
| 版本适配 | `version_understanding` 或 TK 摘要 | 可先截断显示 |
| 风格/强项/风险 | 队伍画像 + notes | 可先用文本摘要 |
| 资料状态 | 是否有画像、三维、TK | 对外卡很重要 |
| 关联 | TK / 图谱 / 详情 | 点击进入 |

前端目标不是做大表格，而是 B站风格的信息卡/横向列表：颜色轻、信息密度够、可扫读。

#### D. 队伍详情

点击横向表某个队伍后，沿用现有：

- 队伍画像
- 三维数据
- 战术笔记
- 版本理解
- 选手列表
- 相关 TK

无需重做，只是作为横向表的详情层。

#### E. TK / 概念图

TK 搜索、编辑、新建能力保留。

概念图入口要提升，不只在右上角放一个按钮。

建议在基本面页放：

```text
概念图
- 全局 TK 图谱
- MSI 图谱
- 当前队伍图谱
- 版本/体系图谱
```

第一期可以先用已有 `/tk-graph/index.html?q=MSI` 这类外链，不强行重写图谱。

### 3.2 盘口页

盘口页不是交易填表。

第一期只做：

```text
盘口分析窗口
├─ 输入框：钧钧写盘口、赔率、临场想法
├─ 关联队伍：可选
├─ 关联基本面：从基本面页带入当前队伍/TK/版本
└─ 可选保存：保存成草稿或 TK，不强迫结算
```

明确不做：

- 不把命中率放主展示。
- 不强制赢输结算。
- 不要求每场记录。
- 不做“按字段填完才能分析”的交易表单。

## 4. 接口怎么套

### 4.1 需要保留的接口

这些继续用：

```text
GET /api/teams
GET /api/team-3d/{team}
PUT /api/team-3d/{team}
GET /api/players?team=
GET /api/wiki/team/{team}
GET /api/profile-full/{team}
GET /api/tk/search
POST /api/tk
DELETE /api/tk/{filename}
GET /api/schedules?event=MSI
GET /api/trades / POST /api/trades  // 只给盘口页可选保存用
```

### 4.2 建议新增的聚合接口

为了前端一次做好，不要让浏览器对每个队伍 N 次请求，建议后端加三个轻量聚合接口。

#### 1. `GET /api/fundamentals/teams?scope=all|lpl|lck|msi`

用途：队伍横向表。

返回示例：

```json
{
  "teams": [
    {
      "short_name": "T1",
      "name": "T1",
      "region": "LCK",
      "has_profile": true,
      "has_3d": true,
      "has_tk": true,
      "dim_1_name": "优势局习惯",
      "dim_1_value": "A",
      "dim_2_name": "劣势局习惯",
      "dim_2_value": "B+",
      "dim_3_name": "胜负手位置",
      "dim_3_value": "中野辅",
      "version_summary": "中野辅节奏、转线、BO稳定性",
      "notes_summary": "摘要...",
      "data_quality": "完整"
    }
  ]
}
```

其中外卡队伍可以返回：

```json
"data_quality": "资料不足"
```

#### 2. `GET /api/fundamentals/msi`

用途：MSI 研究视角，不给用户看赛程表。

返回：

```json
{
  "event": "MSI",
  "teams": [...],
  "regions": {
    "LPL": 2,
    "LCK": 2,
    "INTL": 8
  },
  "missing_profiles": ["TLAW", "KC"],
  "key_topics": ["BO5稳定性", "外卡未知量", "版本适配"]
}
```

实现上可以直接基于 `/api/teams` + `/api/schedules?event=MSI` + wiki/3d 存在性检查。

#### 3. `GET /api/fundamentals/tk?scope=version|msi|team&team=T1`

用途：给基本面页拿 TK 摘要。

第一期也可以先复用 `/api/tk/search`，不一定马上新增。

### 4.3 盘口页接口

第一期不用新表。

可以复用：

```text
POST /api/trades
GET /api/trades
```

但前端文案改成“盘口草稿 / 分析记录”，不要叫交易记录。

如果要更干净，第二期再新增：

```text
POST /api/market-notes
GET /api/market-notes
```

## 5. 前端怎么改

当前前端是 `index.html + src/main.js`，无框架，B站粉蓝风格。

### 5.1 风格延续

继续沿用当前：

- B站粉蓝主色：`#FB7299` / `#00AEEC`
- 白底轻卡片
- 等宽字体做数据标签
- 圆角、轻边框、少量徽标
- 命令栏 / 快捷 chip
- 单页工作台

不换技术栈，不换视觉底座。

### 5.2 页面结构调整

从当前：

```text
队伍画像栏 + MSI赛程卡 + 交易记录卡 + 三维卡 + TK卡
```

改成：

```text
顶部：品牌 / 基本面-盘口页签 / 搜索 / 队伍筛选

基本面页：
├─ MSI 国际赛环境卡
├─ 版本 / 体系卡
├─ 队伍横向表
├─ 概念图 / 分类图
└─ 队伍详情抽屉或右栏

盘口页：
├─ 盘口分析输入窗口
├─ 当前引用的基本面依据
└─ 可选草稿列表
```

### 5.3 现有模块迁移

| 当前模块 | 改法 |
|---|---|
| MSI 工作台 | 改成“MSI 国际赛环境”，不显示完整赛程表，只显示队伍池/资料缺口/赛区差异 |
| 交易记录 | 移到“盘口”分页，弱化统计和结算 |
| 三维数据 | 从单队卡变成横向表字段 + 详情编辑 |
| 队伍画像 | 保留为详情层 |
| TK 搜索 | 保留，并和基本面卡片互相联动 |
| 概念图按钮 | 提升为基本面主区 |

## 6. 能不能一次落实到位

可以一次做成一个可用版本，但建议不要一次塞太多“智能分析”。

### 一次落地范围建议

一次做到这些：

1. 新增后端聚合接口：
   - `/api/fundamentals/teams`
   - `/api/fundamentals/msi`
2. 前端新增页签：
   - `基本面`
   - `盘口`
3. 基本面页完成：
   - MSI 国际赛环境卡
   - 队伍横向表
   - 队伍详情联动
   - TK/概念图入口
4. 盘口页完成：
   - 输入窗口
   - 引用当前队伍/基本面
   - 可选保存草稿，先复用 `/api/trades`
5. 原有 B站粉蓝风格延续。
6. 跑构建和后端编译验证。

### 不建议第一期做的

- 不做复杂模型自动总结每队风格。
- 不做全自动盘口建议。
- 不做命中率统计主面板。
- 不改数据库大结构。
- 不重写 TK 图谱服务。
- 不把日报搬进前端。

## 7. 风险和处理

| 风险 | 影响 | 处理 |
|---|---|---|
| 外卡队伍资料不足 | 横向表空白 | 明确显示“资料不足 / 待补”，不伪装完整 |
| 单页原生 JS 继续变大 | 维护成本上升 | 第一版仍可接受，后续再拆文件 |
| 队伍横向接口聚合慢 | 页面加载慢 | 后端一次 SQL + 文件存在性检查，摘要截断，避免逐队前端请求 |
| 交易接口名字仍叫 trades | 语义有偏差 | 第一版前端改文案，第二期再改 `market-notes` |
| 概念图是外链 | 体验不够一体 | 先保留新标签页，后续 iframe 或嵌入 |

总体风险可控。

## 8. 验收口径

完成后至少验证：

```bash
cd /home/ubuntu/xiaoxue-web
npm run build
python3 -m py_compile main.py
curl http://127.0.0.1:8880/api/fundamentals/teams?scope=msi
curl http://127.0.0.1:8880/api/fundamentals/msi
```

页面验收：

- 打开 8880 正常显示。
- 默认进入“基本面”。
- 能看到 MSI 国际赛环境，不是赛程表主展示。
- 能看到队伍横向表，LPL/LCK/INTL 有区分。
- 外卡队伍资料不足会明确标注。
- 点击队伍能查看画像/三维/TK。
- “盘口”分页能输入内容，不强迫结算。
- 命中率/赢输统计不在主流程。

## 9. 最终建议

建议可以做，而且可以一次做到可用成品。

执行顺序：

```text
1. 后端加 fundamentals 聚合接口
2. 前端改成 基本面 / 盘口 两页
3. 基本面页做 MSI 环境 + 队伍横向表 + TK/概念图
4. 盘口页迁移原交易能力但弱化成输入窗口
5. 构建、编译、curl 验证
6. 给钧钧看 8880 成品
```

这次不是加新系统，是把已有系统重新排成钧钧真正会用的形态。
