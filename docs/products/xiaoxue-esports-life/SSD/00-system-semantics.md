# 小雪电竞人生 — 系统语义设计文档 (SSD)

> 基于小雪电竞 Wiki、v1.0 工作台与 2026-06-29 横向基本面改造更新
> 版本：1.1 | 2026-06-29

---

## 1. 项目一句话定义

> 小雪电竞人生是 LOL 横向基本面工作台：把版本/体系、LPL/LCK 队伍、MSI 国际赛环境、外卡资料缺口、队伍/选手画像、三维数据、TK 与概念图组织成长期可查的基本面系统；盘口另起分页，只给钧钧自己分析。

---

## 2. 核心语义

| 术语 | 定义 | 禁用替代 |
|------|------|----------|
| 横向基本面 | 按队伍/版本/体系/赛区横向比较 LOL 基本面 | 赛程表、资料堆 |
| MSI 国际赛环境 | 研究 MSI 队伍池、跨赛区强弱、外卡未知量、版本适配、BO 稳定性和资料缺口 | MSI 赛程页 |
| 三维数据 | 队伍在优势局、劣势局、胜负手位置等维度上的结构化判断 | 简单评分 |
| TK | 可复用的战术知识、版本理解、队伍模式、判断依据 | 灵感、闲聊记录 |
| 队伍画像 | 对队伍核心方程、优势场景、崩溃条件、版本适配的结构化描述 | 简介 |
| 选手画像 | 对选手英雄池、风格、节奏、风险点的结构化描述 | 八卦资料 |
| 盘口分析 | 钧钧自己写盘口、赔率、临场想法，可引用基本面依据 | 交易填空题 |
| 日报 | 对当日赛程、结果、动态、知识变化的查询沉淀 | 泛新闻、前端赛程页 |

## 3. 工作台四层语义

### 基本面层：平时怎么看懂队伍和版本

```text
功能：版本/体系、队伍横向表、队伍画像、选手画像、TK、概念图。
用户动作：横向扫队伍 → 点开详情 → 查依据。
数据展位：基本面页。
语义：小雪前端主线，不是赛程 App。
```

### MSI 环境层：当前国际赛怎么研究

```text
功能：MSI 队伍池、赛区分布、INTL/外卡资料缺口、跨赛区强弱、BO 稳定性。
用户动作：切到 MSI scope → 看哪些队伍完整、哪些待补、哪些赛区差异明显。
数据展位：MSI 国际赛环境卡、队伍横向表。
语义：赛程只作为队伍池和对局背景，不作为用户看表入口。
```

### 依据层：为什么这样看

```text
功能：TK 搜索、战术概念、B站/微信/用户记录、日报沉淀、概念图。
用户动作：搜索关键词 → 展开条目 → 引用到基本面或盘口分析。
数据展位：TK 面板、概念图/分类图。
语义：判断必须有可追溯依据，不靠临场感觉。
```

### 盘口层：钧钧自己怎么想

```text
功能：盘口输入窗口、基本面引用、可选保存。
用户动作：写盘口、赔率、临场想法 → 可选保存。
数据展位：盘口页。
语义：不是交易记录系统，不强迫结算，不把命中率作为主功能。
```

## 4. 四层关系图

```text
基本面 ──提供长期结构──→ MSI 环境
MSI 环境 ──指出队伍池/资料缺口──→ 横向队伍表
依据层 ──提供来源和概念──→ 基本面 / 交易页
交易层 ──钧钧自己手动分析──→ 可选保存 / 后续沉淀
日报 ──承接每日比赛──→ 不占前端主流程
```

---

## 5. 输出结构

每次重要展示或沉淀至少包含：

| 部分 | 内容 | 数据来源 |
|------|------|----------|
| 对象 | 队伍、赛区、赛事、版本、选手 | teams / rosters / schedules |
| 当前状态 | 三维、画像摘要、资料完整度 | team_3d_data / Wiki |
| 判断依据 | TK、战术概念、历史模式 | RAG / Wiki / B站 / 微信 / 用户记录 |
| 资料缺口 | 外卡/INTL 是否缺画像、缺三维、缺 TK | fundamentals 聚合接口 |
| 盘口输入 | 钧钧自己的盘口想法 | 盘口页输入 / 可选保存 |

---

## 6. 文案原则

- 用“横向基本面”，不要写成“赛程查询”。
- 用“MSI 国际赛环境”，不要写成“MSI 赛程页”。
- 用“盘口分析窗口”，不要写成“交易记录主流程”。
- 用“资料不足 / 待补”，不要伪装外卡队伍资料完整。
- API 失败时直接显示失败，不写“已生成”。
- 小雪是电竞人生产品，不承担结衣知行合一的个人反馈语义。
