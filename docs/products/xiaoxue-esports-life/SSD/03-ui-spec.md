# 小雪电竞人生 — 界面规格 (UI Spec)

> 版本：2.0 | 2026-07-09
> 口径：UI 服务电竞交易辅助系统；当前页面仍以 LOL 第一项目为主。

---

## 1. 设计系统

### 设计 DNA

- 延续 B站粉蓝：粉 `#FB7299` + 蓝 `#00AEEC`。
- 白底轻卡片、浅灰背景、轻边框、数据标签。
- 信息密集但不乱，每个像素服务判断。
- 等宽或半等宽字体用于队伍编码、TS、盘口、数据标签。
- 编辑态需要明确激活反馈。
- 桌面优先，1920px 信息密度。

### 禁用

- 不做赛程 App 样式。
- 不把盘口页做成自动交易填空表。
- 不把命中率/赢输统计放主流程。
- 不做移动端优先布局。
- 不把日报卡片做成总纲中心。

```text
Top Bar：小雪 + 日报 / 工作台
工作台二级导航：队伍资料 / 当前赛事 / TK 资料库

  隐藏辅助：market_notes 盘口记录
```

## 4. 从主导航移出的内容

- 今日赛前和日报副本。
- BP 输入与阵容分析。
- 小雪聊天。
- 交易预案生成。
- 三层对照。
- 阶段整理与复盘入口。
- 盘口记录；保留为 `market_notes` 隐藏辅助。

这些能力不删除；动态能力统一从小雪对话进入。

## 5. 命令栏

命令栏只承担“小雪对话”的快捷接入，不再成为另一套页面导航。

输入示例：

| 指令 | 行为 |
|---|---|
| `看TES` | 由小雪查询并解释 TES 资料；需要长期浏览时指向队伍资料 |
| `TK 野区` | 由小雪检索 TK |
| `制定当前赛事交易预案` | 进入大周期预案对话 |
| `BP 出来了` / `蓝色方...红色方...` / `只看阵容` | 在对话中触发 `lol-lineup-analysis` |
| `复盘刚才这场` | 在对话中读取原始记录并开始复盘 |

## 6. 队伍资料：长期依据层

### MSI 国际赛环境卡

展示：

- MSI 队伍池数量。
- 资料缺口数量。
- 赛区分布：LPL / LCK / INTL。
- 关键主题：跨赛区强弱、外卡未知量、版本理解差、BO 稳定性、资料缺口。
- 文案明确：国际赛环境研究，不是赛程表。

### 队伍横向表

| 字段 | 说明 |
|---|---|
| 队伍 | short_name / name |
| 赛区 | LPL / LCK / INTL |
| TS | mu、sigma、TS、risk_gap |
| 优势 | dim_1_value |
| 劣势 | dim_2_value |
| 胜负手 | dim_3_value |
| 版本/风格摘要 | version_summary 或 notes_summary |
| 资料 | 完整 / 部分 / 资料不足 |

交互：

- scope chips：MSI / LPL / LCK / INTL / ALL。
- 点击队伍加载左侧画像栏、三维详情、队伍 TK。
- 资料不足必须明显标注。

### 队伍资料

展示：

- 画像。
- 三维。
- 队伍 TK，其中 `type=trading_note` 的交易 TK 单独置顶。
- 边界：不建立队伍版本栏目；全局版本只在版本理解中展示。

## 7. 日报：每日默认入口

今日内容卡读取 `/api/daily-content`，只读本地白名单文件：

| 内容 | 说明 |
|---|---|
| 昨日比赛复盘 | 昨日判断与实际结果的校准摘要 |
| 当前交易预案摘要 | 只引用已确认预案；没有时显示“尚未制定” |
| 今日赛程 | 只显示真实已确认对局 |
| 当前版本要点 | 当前可用版本事实与风险 |
| 今日逐场分析 | 每场只写既有赢法、对局关系、赛前倾向和不确定性 |
| 最近相关 TK | 与今日比赛相关的正式 TK |
| 缺失与风险 | 数据、资料或自动化异常必须显式显示 |

UI 边界：

- 只读存在状态、摘要、路径。
- `/api/daily-content` 同时返回当天已确认赛程与 `day_state`：`no_matches / content_missing / ready`。
- `no_matches` 明确显示“今日无比赛”，不要求生成赛前日报，也不增加前端复盘入口。
- `content_missing` 表示当天有真实比赛但赛前内容未齐；只能显示真实赛程和缺失提示，不能拿旧比赛或静态样例顶替。
- 接口失败或状态不明确时隐藏比赛卡，不猜测“休赛日”。
- 不保存。
- 不发布。
- 不覆盖 Wiki 正式日报。
- 不把日报样式当总纲中心。

## 8. 小雪对话中的 BP 后阵容判断

用途：BP 后补齐蓝红方五位置英雄，并复制给小雪触发 `lol-lineup-analysis` 纯十英雄阵容分析 v7.4。赛前底稿、盘口记录和赛后复盘继续使用各自独立入口，不在这里拼接。

UI 行为：

- 可以整理阵容提示文本。
- 可以复制给小雪。
- 必须明确：纯阵容分析在小雪对话 + `lol-lineup-analysis` Skill，前端只整理十英雄文本。
- 不把纯阵容分析做成静态打分表，也不自动拼接队伍、选手、TS、三维、盘口或赛前想法。
- 不在前端直接调用 LLM 生成自动交易结论。
- 十英雄不完整时必须显示待补，不能生成分析提示。

### 交易时 / 纯十英雄阵容分析 v7.4 轻入口

工作台不提供 BP 页面或卡片。真实用法是：钧钧在小雪对话中直接给出蓝红方十英雄，小雪触发 `lol-lineup-analysis`。

前端可以辅助复制：

| 字段 | 说明 |
|---|---|
| 蓝色方 | 上 / 野 / 中 / 下 / 辅五个英雄 |
| 红色方 | 上 / 野 / 中 / 下 / 辅五个英雄 |

前端输出：

```text
小雪，只看十英雄分析这把。
蓝色方：上 / 野 / 中 / 下 / 辅
红色方：上 / 野 / 中 / 下 / 辅
```

小雪收到后进入 `lol-lineup-analysis` v7.4，默认等经济、正常发育、同等操作水平，先输出固定30分钟的偏小/偏大/放弃建议，再输出总体粗略比例、四阶段强度、双方比赛剧本、节奏、人头高/中/低倾向、走势和观赛信号；不输出宽时间范围，终结与拖延信号冲突时直接放弃。需要完整交易判断时，钧钧另行明确调用 `junjun-trading-system`；前端不把两条链拼成一个模板。

## 9. 隐藏盘口记录辅助

盘口记录不在主导航，也不主动催促填写。钧钧按需打开后，可把原始文本保存到 `market_notes`。赛后由钧钧回到小雪对话，让小雪读取这些原始记录并共同复盘。

字段：

| 字段 | 说明 |
|---|---|
| 项目 | 当前 LOL，可预留 CS / Valorant / 其他 |
| 比赛 / 对象 | 可手填，也可从当前队伍带入 |
| 时间 | 可选 |
| 盘口 / 方向 | 钧钧手写 |
| 大小 / 总局数 | 钧钧手写 |
| 赔率 / 比分 | 钧钧手写 |
| 我的判断 / 市场分歧点 / 不碰项 | 主文本，保留交易时原始判断 |
| 结果 | 未结算 / 赢 / 输 / 走水 / 放弃；复盘辅助字段，不代表自动交易结论 |
| 备注 / 复盘原始记录 | BP、首发、临场、赛后复盘；结果当前以固定行 `结果：...` 写入这里，不改数据库结构 |

操作：

- 保存到 `/api/market-notes`。
- 删除 market_notes。
- “记到队伍”写入 `/api/team-trading-notes/from-text`，归属队伍 TK。
- 草稿可手动转 TK。
- 当前列表加载最近 `market_notes` 原始记录，并提供轻量本地筛选：最近 7 天、最近 30 天、全部最近记录、队伍/比赛关键词、结果。
- 结果先写入 `review` 固定行 `结果：未结算/赢/输/走水/放弃`；不新增 schema，后续需要正式统计时再做结构化迁移。
- 前端不显示“开始复盘”，不弹复盘问卷，也不在页面里完成 `review-preview → confirmed writeback`；这些后台能力只供小雪对话复盘链使用。

不展示：

- 不把旧 `/api/trades/stats` 作为主面板。
- 不强迫结算。
- 不展示命中率作为主流程。
- 不输出自动买卖。

## 11. TK 与概念图

- 顶部提供独立“TK资料库”：默认最新优先，支持今天、最近 7 天、最近 30 天、按月份、全部，以及队伍/关键词筛选。
- 列表以结论和正文预览为主，日期持续可见；来源、文件名和导入字段留在后台。
- 每次显示 30 条并继续加载更早内容；“阅读全文”打开宽版阅读层，按结论/因果/推演分段且不截断正文。
- TK 搜索保留展开、编辑、新建、删除。
- TK 作为基本面、单场判断、市场对照和复盘校准的依据库。
- 概念图可嵌入或新标签打开。
- 交易 TK 本质是带结构标签的队伍 TK；队内和日报按比赛优先显示。

## 12. 空状态与失败状态

| 场景 | 展示 |
|---|---|
| 未选择队伍 | 默认展示基本面页和 MSI 横向表 |
| 今日无已确认比赛 | 显示“今日无比赛”，隐藏旧赛程与赛前内容文件卡，不增加前端复盘入口 |
| 当天有比赛但今日内容文件缺失 | 明确显示比赛存在但日报/赛前卡/交易判断日报尚未生成，不用静态样例替代 |
| 今日内容接口或状态异常 | 不展示旧赛程，提示当前无法确认并建议稍后重试 |
| 队伍资料失败 | “队伍资料加载失败” |
| MSI 环境失败 | “MSI 环境数据不可用” |
| 队伍数据失败 | “队伍数据加载失败” |
| TK 搜索失败 | “TK 搜索服务不可用” |
| 三维保存失败 | “保存失败，未写入数据库” |
| market_notes 保存失败 | “保存失败，稍后重试” |
| 队伍交易备注队伍不明确 | “队伍未确认，未写入正式 TK” |
| 数据不足 | “资料不足”或“暂不推荐” |
