# 小雪电竞人生 — 界面规格 (UI Spec)

> 基于当前小雪工作台与 2026-06-29 横向基本面改造更新
> 版本：1.1 | 2026-06-29
> 设计方向：延续 B站粉蓝工作台风格

---

## 1. 设计系统

### 设计 DNA

- 延续 B站粉蓝：粉 `#FB7299` + 蓝 `#00AEEC`。
- 白底轻卡片、浅灰背景、轻边框、数据标签。
- 信息密集但不乱，每个像素服务判断。
- 等宽或半等宽字体用于队伍编码、数据标签。
- 编辑态需要明确激活反馈。
- 桌面优先，1920px 信息密度。

### 禁用

- 不换视觉底座。
- 不做赛程 App 样式。
- 不把盘口页做成交易填空表。
- 不把命中率/赢输统计放主流程。
- 不做移动端优先布局。

## 2. 顶层布局

```text
┌────────────────────────────────────────────────────────────┐
│ Top Bar：小雪工作台 + 基本面/交易页签 + 队伍选择 + 图谱按钮 │
├────────────────────────────────────────────────────────────┤
│ Command Bar：基本面 / MSI研究 / MSI概念图 / 交易页 / TK版本理解 │
├──────────────┬─────────────────────────────────────────────┤
│ 队伍详情栏    │ 基本面页：MSI环境 / 队伍横向表 / TK / 3D详情   │
│              │ 交易页：交易分析输入窗口 / 可选保存           │
└──────────────┴─────────────────────────────────────────────┘
```

基准：1920px 桌面端。

---

## 3. 顶部栏

### 元素

- 品牌：小雪工作台。
- 一级页签：`基本面` / `盘口`。
- 队伍选择器：LPL / LCK / INTL 分组。
- 工具按钮：关联图。
- 状态：服务状态、时间。

### 交互

- 默认进入 `基本面`。
- 点击 `盘口` 切到盘口分析页。
- 选择队伍后刷新画像、三维、TK。

## 4. 命令栏

### 快捷 chips

| Chip | 行为 |
|------|------|
| 基本面 | 回到基本面页 |
| MSI研究 | 加载 MSI scope 的横向基本面 |
| MSI概念图 | 打开 `/tk-graph/index.html?q=MSI` |
| 盘口 | 切到盘口页 |
| TK版本理解 | 搜索版本理解相关 TK |

### 输入示例

| 指令 | 行为 |
|------|------|
| `看TES` | 切换到 TES |
| `TK 野区` | 搜索野区相关 TK |
| `MSI` | 切到基本面页，加载 MSI 横向表并搜索 MSI TK |
| `盘口` | 跳转盘口分析页 |
| `改优势局习惯 ...` | 修改三维字段 |

## 5. 基本面页

### 5.1 MSI 国际赛环境卡

展示：

- MSI 队伍池数量。
- 资料缺口数量。
- 赛区分布：LPL / LCK / INTL。
- 文案明确：国际赛环境研究，不是赛程表。

### 5.2 队伍横向表

字段：

| 字段 | 说明 |
|------|------|
| 队伍 | short_name |
| 赛区 | LPL / LCK / INTL |
| 优势 | dim_1_value |
| 劣势 | dim_2_value |
| 胜负手 | dim_3_value |
| 版本/风格摘要 | version_summary 或 notes_summary |
| 资料 | 完整 / 部分 / 资料不足 |

交互：

- scope chips：MSI / LPL / LCK / INTL / ALL。
- 点击队伍加载左侧画像栏和三维详情。
- 资料不足用红色提示，部分资料用橙色提示，完整用绿色提示。

### 5.3 概念图 / TK

- 基本面页提供 MSI 概念图入口。
- TK 搜索保留展开、编辑、新建、删除。
- TK 作为基本面依据库，不只是孤立搜索框。

## 6. 队伍详情 / 三维面板

### 字段

| 字段 | 可编辑 | 说明 |
|------|--------|------|
| 优势局习惯 | 是 | dim_1 |
| 劣势局习惯 | 是 | dim_2 |
| 胜负手位置 | 是 | dim_3 |
| 战术笔记 | 是 | notes |
| 版本理解 | 是 | version_understanding |
| 更新时间 | 否 | updated_at |

### 交互

- 点击横向表队伍后加载。
- 修改后出现 dirty 状态。
- 保存调用 `PUT /api/team-3d/{team}`。
- 保存失败显示“保存失败”。

---

## 7. 盘口页

### 定位

盘口页是钧钧自己的分析窗口，不是交易填空题。

### 字段

| 字段 | 说明 |
|------|------|
| 比赛 / 对象 | 可手填，也可从当前队伍带入 |
| 时间 | 可选 |
| 方向 | 钧钧自己的方向 |
| 大小 / 比分 | 可选 |
| 理由 | 钧钧自己的盘口分析 |
| 盘口备注 | 可选，不强制 |

### 操作

- 可选保存。
- 可从当前队伍带入基本面上下文。
- 可删除旧草稿。

### 不展示

- 不把总记录、已结算、赢、输、走水、命中率作为主流程展示。
- 不强迫每笔结算。

---

## 8. 空状态与失败状态

| 场景 | 展示 |
|------|------|
| 未选择队伍 | 默认展示基本面页和 MSI 横向表 |
| 横向基本面失败 | “横向基本面加载失败” |
| MSI 环境失败 | “MSI 环境数据不可用” |
| 队伍数据失败 | “队伍数据加载失败” |
| TK 搜索失败 | “TK 搜索服务不可用” |
| 三维保存失败 | “保存失败，未写入数据库” |
| 盘口保存失败 | “保存失败，稍后重试” |
