# 小雪电竞人生｜分析师入口说明面板内容稿（历史归档）

> **历史归档，禁止作为当前前端文案或行为正源。** 本稿把队伍、TS、市场与阵容混在同一入口，已被 2026-07-16 的分流口径取代。当前纯十英雄阵容入口只复制蓝红方十英雄并调用 `lol-lineup-analysis` v7.3；明确交易请求另行调用 `junjun-trading-system`。当前 UI 口径见 `SSD/03-ui-spec.md`。

> 原用途：给旧前端“分析师入口 / 单场分析入口”提供文案。以下内容仅供追溯，不得直接取用。

## 1. 面板定位

分析师入口不是赛程表，也不是自动交易按钮。

它服务三类场景：

1. 赛前单场：钧钧输入一场比赛，系统先调出双方基本面、TS 单场底表、盘口观察点和 BP 待补字段。
2. BP 后阵容分析：阵容锁定后，钧钧补蓝红方、五个位置和英雄，系统按单场 / BP 分析链拆阵容怎么赢、风险在哪、什么时候兑现。
3. 赛后复盘：比赛结束后，钧钧补赛果、关键局、关键转折，系统把赛前判断、BP 结构和实际走势对齐，沉淀复盘字段。

面板文案建议：

“小雪可以陪你做单场分析：赛前先看基本面和波动，BP 后拆阵容链路，赛后复盘判断有没有兑现。盘口只做观察，不自动给交易结论。”

## 2. 用户输入什么

### 2.1 最小输入

钧钧只需要先输入一场比赛：

- 比赛日期：例如 2026-07-05
- 对局：例如 TSW vs TES、HLE vs G2
- 赛制 / 阶段：例如 MSI 季中赛 BO5

可选输入：

- 当前盘口 / 赔率变化
- 钧钧自己的初始判断
- 特别想看的点：强弱差、爆冷空间、BP 风险、资源团、盘口分歧

### 2.2 BP 后补充输入

BP 出来后再补，不要在赛前假装已有：

- 蓝方 / 红方
- 双方上野中下辅五个英雄
- 第一轮 ban / pick 重点
- 第二轮补强或漏洞
- 首发确认与临场换人
- 蓝红方胜率、版本强势英雄交换、特殊体系

### 2.3 赛后复盘输入

赛后才补，不要在赛前预写成事实：

- 最终比分与每局胜负
- 每局蓝红方与阵容
- 关键资源团：小龙、巢虫 / 先锋、大龙、远古龙
- 关键转折：前 10 分钟线权、14 分钟经济、第一轮资源交换、团战失误
- 盘口实际变化与钧钧赛前判断是否偏离
- 需要沉淀到 TK / 日报 / 队伍画像的结论

## 3. 系统输出什么

### 3.1 赛前输出

系统先输出“够用版单场底稿”：

1. 对局确认：日期、时间、队伍、赛制、赛事阶段。
2. 单场 TS 底表：双方 mu、sigma、TS、risk_gap、资料置信度。
3. 强弱与波动：谁均值更高、谁波动更大、TS 下界差说明什么。
4. 盘口观察：只提示市场可能高估 / 低估 / 简化的点，不写买谁。
5. BP 待补：列出阵容出来后必须补的字段。
6. 临场破相条件：哪些早期信号会让赛前判断需要降权。

### 3.2 BP 后输出

系统在阵容确认后输出“单场 / BP 分析”：

1. 阵容确认：先复述蓝红方和五个位置，避免错阵容分析。
2. 控制量化：开团、反开、点控、AOE 控、资源团控制从哪里来，几级 / 几件套后成型。
3. 基本面 / TS：把赛前 TS、队伍画像、三维和当前阵容执行难度合并看。
4. 线权时间边界：1-6 级、第一轮资源、14 分钟、一二件套、三件套后的主动权变化。
5. 核心链：一句话说清这套阵容怎么赢，以及链路断点。
6. 24 场景定位：强开、反开、poke、滚雪球、后期大核、边线牵扯、资源团阵地等。
7. 风险点：区分阵容结构风险和队伍状态风险。
8. 交易边界：只提示市场分歧和风险，不替钧钧下交易结论。

### 3.3 赛后输出

系统在赛后输出“复盘沉淀”：

1. 赛前判断回看：强弱差、波动点、盘口观察是否被验证。
2. BP 兑现度：阵容核心链是否成立，断点出现在哪里。
3. 时间线复盘：前 10 分钟、第一轮资源、14 分钟、一二件套、关键龙团。
4. 队伍画像更新：哪些内容应该进入三维、画像或 TK。
5. 日报 / Wiki 摘要：压缩成钧钧之后能复用的一段话。
6. 禁止自动交易结论：赛后可以记录盘口观察与判断偏差，但不生成“下次自动买 / 卖”的指令。

## 4. 赛前可用字段

赛前已经可以使用的字段：

- 比赛日期、时间、赛事、阶段、BO 几
- 队伍名称、赛区、基础画像
- MSI TS 字段：mu、sigma、TS、risk_gap
- 队伍三维：优势局、劣势局、胜负手、资料完整度
- 赔率种子或市场位置参考
- 日报上下文、TK / 概念图相关条目
- 盘口观察点：市场是否只看强队名气、是否低估高波动方、是否把下界优势一次性打满

注意：赛前没有 BP、没有实际对局过程、没有赛后结果。面板必须把这些标成“待补”，不能写成已经发生。

## 5. BP 后补字段

BP 后再补以下字段：

- 蓝红方
- 上野中下辅英雄
- ban 位重点
- 阵容标签：强开、反开、poke、边线、后期大核、前中期滚雪球、资源团阵地
- 控制来源：点控、AOE 控、开团、反开、视野控制
- 伤害结构：AD/AP、持续 / 爆发、前中后期曲线
- 前排与容错
- 线权与资源节奏
- 阵容核心链与断点
- BP 后盘口变化，只作为观察字段

## 6. 赛后复盘字段

赛后再补以下字段：

- 最终比分、每局胜负、每局蓝红方
- 每局 BP 与阵容标签
- 前 10 分钟线权和野区情况
- 第一轮资源交换
- 14 分钟经济 / 塔皮 / 先锋或小龙收益
- 中期关键团战、视野失误、资源决策
- 赛前 TS / 波动判断是否兑现
- BP 后核心链是否成立
- 盘口观察是否偏离基本面
- 可沉淀到队伍画像、三维、TK、日报的内容

## 7. 示例：TSW vs TES

赛前输入示例：

- 日期：2026-07-05
- 对局：TSW vs TES
- 赛制：MSI 季中赛 BO5

赛前可输出：

- 单场 TS 底表：TSW mu 22.8 / sigma 6.0 / TS 4.8 / risk_gap 18.0；TES mu 26.9 / sigma 0.5 / TS 25.5 / risk_gap 1.4。
- 强弱差：TES 均值和 TS 下界更高，是赛前强弱基准。
- 波动点：TSW 的 sigma 和 risk_gap 明显更大，说明单局扰动空间高，不能只用均值看。
- 盘口观察：先看市场是否只围绕 TES 名气和 LPL 强队热度定价，再看 TSW 受让 / 小局空间有没有被高波动充分计入。
- BP 待补：蓝红方、首发、第一轮 ban/pick、TES 中野资源掌控、TSW 是否拿到主动开团或前中期碰撞阵容。
- 临场破相条件：TES 前两轮资源团处理粗糙、野区被入侵后没有备选节奏，或 14 分钟前线权没有转成先锋 / 小龙收益。

不能写成事实的内容：

- 不假装已知道 TSW vs TES 的 BP。
- 不假装已知道赛果、每局走势或赛后数据。
- 不输出“自动买 TES / 自动买 TSW 受让”这类结论。

## 8. 示例：HLE vs G2

赛前输入示例：

- 日期：2026-07-05
- 对局：HLE vs G2
- 赛制：MSI 季中赛 BO5

赛前可输出：

- 单场 TS 底表：HLE mu 31.8 / sigma 0.8 / TS 29.5 / risk_gap 2.3；G2 mu 26.0 / sigma 4.8 / TS 11.6 / risk_gap 14.4。
- 强弱差：HLE 均值和 TS 下界更高，是更稳定的强弱基准。
- 波动点：G2 的高 sigma 代表上限更依赖版本理解、奇招 BP 和中期决策。
- 盘口观察：先确认市场是否把 HLE 的 LCK 强队身份和 TS 下界优势一次性打满，再看 G2 高波动是否只被当成弱队不稳。
- BP 待补：HLE 是否拿到中野强势碰撞、上路边线压制和资源团先手；G2 是否拿到边线牵扯、开团反开或中期多点节奏。
- 临场破相条件：HLE 上中野前 10 分钟没有压出线权、第一轮小龙 / 先锋交换吃亏，或团前视野连续被 G2 先占。

不能写成事实的内容：

- 不假装已知道 HLE vs G2 的 BP。
- 不假装已知道赛果、每局走势或赛后数据。
- 不输出“自动买 HLE / 自动买 G2”这类结论。

## 9. 面板短文案

标题：

单场分析师入口

说明：

输入一场比赛，小雪先给你赛前基本面、TS 波动、盘口观察和 BP 待补字段；BP 出来后再补蓝红方与五个英雄，拆阵容核心链、时间边界和风险点；赛后可以继续复盘判断是否兑现。盘口只做观察，不生成自动交易结论。

输入提示：

输入日期 + 对局即可开始，例如：2026-07-05 TSW vs TES。BP 出来后补蓝方 / 红方 / 五个位置；赛后再补比分、关键资源团和转折点。

输出提示：

输出包含单场底表、强弱与波动、盘口观察、BP 待补、阵容链路、风险点和复盘沉淀。没有 BP 或赛后数据时，会明确标记为待补，不假装已有。

## 10. 禁止自动交易结论

必须在面板中保留这条边界：

小雪可以输出盘口观察、市场分歧、风险提示和赛后偏差复盘，但禁止自动交易结论。不得生成“买谁、卖谁、必穿、必中、自动跟单、自动结算、下次按规则自动执行”等内容。最终交易判断由钧钧自己定。
